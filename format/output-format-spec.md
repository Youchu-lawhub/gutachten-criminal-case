## 二、9级标题字体矩阵

| 级别 | Markdown | 字体 | 字号 | 加粗 | 行距 | 说明 |
|------|----------|------|------|------|------|------|
| 报告标题 | `# 标题` | 黑体 | 小三(15pt) | 是 | 1.25 | 全文仅一个 |
| 部分层 | `## 第X部分` | 黑体 | 小三(15pt) | 是 | 1.25 | ≥2参与人才加 |
| 一级·罪名 | `### A的行为可能构成XX罪 ✓/✗` | 黑体 | 小三(15pt) | 是 | 1.25 | 鉴定句式 |
| 二级·大纲/正文 | `#### （一）大纲 / （二）正文` | 黑体 | 四号(14pt) | 是 | 1.25 | 强制双H2 |
| 三级·阶层 | `##### 1. 构成要件符合性` | 黑体 | 小四(12pt) | 是 | 1.25 | 三阶层体系 |
| 四级·要件 | `###### (1) 客观构成要件` | 宋体 | 小四(12pt) | 加粗 | 1.25 | 要件分组 |
| 五级·子要件 | `####### ① 行为主体` | 宋体 | 小四(12pt) | 否 | 1.25 | 单要素拆分 |
| 六~七级 | 加粗段头 `A. / a.` | 宋体 | 小四(12pt) | 加粗段头 | 1.25 | 学说展开 |
| 八~九级 | 加粗段头 `(a) / (aa)` | 宋体 | 小四(12pt) | 加粗段头 | 1.25 | 极端嵌套 |

### 使用说明

1. **标题深度原则**：常规论证使用至第五级即可；核心争议点允许展开至第七~九级。
2. **未遂犯/共犯**等特殊犯罪形态可能增加阶层（如"前置检验"），不影响标题级别分配。
3. **大纲/正文双H2结构**：每一罪名（### 级）下必须包含 `#### （一）大纲` 和 `#### （二）正文` 两个H2子节。

## 三、引用块样式

- **字体**：楷体
- **字号**：小四 (12pt)
- **样式**：左侧灰色竖线（Markdown blockquote `>`）
- **内容结构**：法条全文 + MCP校验状态标签

### 校验状态标签

```
> 刑法第二百三十二条　故意杀人的，处死刑、无期徒刑或者十年以上有期徒刑……
> [校验2026-06-15, 现行有效]
```

三种状态：
- `[校验YYYY-MM-DD, 现行有效]` — 校验通过
- `[⚠ 校验失败]` — MCP未能检索到匹配法条
- `[⚠ 语义差异]` — 检索到法条但文本不完全一致

---

## 四、段首缩进

- 正文段落首行缩进：使用 U+3000（全角空格）× 2
- Markdown源文件中直接输入两个全角空格
- 引用块内部同样适用

---

## 五、来源标识符

仅允许以下四种脚注来源标识符：

| 标识符 | 含义 |
|--------|------|
| `[评注]` | 法律评注/教科书引用 |
| `[案例]` | 司法案例引用 |
| `[联网]` | 网络检索来源 |
| `[KB:]` | 知识库检索来源（冒号后接KB名称） |

### 脚注格式

```
正文内容。^[来源标识符 作者/案号, 具体出处, 页码/段落]
```

## 六、文件命名规则

```
{案件简称}+鉴定式刑法案例分析报告.md
{案件简称}+鉴定式刑法案例分析报告.docx
```

- 案件简称：由用户提供或从案情中提取（≤10字）
- 示例：`张三故意伤害案+鉴定式刑法案例分析报告.md`

---

## 七、字数限制

- **上限**：≤ 30,000 字（含脚注）
- 超出时优先压缩非争议要件的论证篇幅
- 脚注计入总字数

---

## 八、格式转换优先级

```
1. cx-md2word --preset=legal    （首选）
2. docx skill                    （备选）
3. Pandoc fallback               （兜底）
   pandoc input.md -o output.docx --reference-doc=template.docx
```

### 转换检查清单

- [ ] 标题层级正确映射为Word标题样式
- [ ] 引用块转换为楷体格式
- [ ] 页边距/行距与本规范一致
- [ ] 页码居中
- [ ] 段首缩进保留

---

## 九、其他排版规则

1. **中西文间距**：中文与英文/数字之间自动添加半角空格（由转换工具处理）
2. **法条编号**：使用中文数字（第二百三十二条），不使用阿拉伯数字
3. **参与人代称**：首次出现使用全名，后续使用单字代称（如"甲""乙"或姓氏）
4. **结论标记**：每一检验步骤结尾使用 ✓（肯定）或 ✗（否定）标记结论
