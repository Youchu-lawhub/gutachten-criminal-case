# 鉴定式刑法案例写作规范

---

## 一、两种语言模式

### 1.1 鉴定式

#### 1.1.1 核心特征

鉴定式是以**假设为起点**、以**系统验证为过程**、以**结论为终点**的论证模式。其本质是"先问后答"。

**逻辑结构**：

```
假设 → 检验 → 结论
```

**语言标志**：

| 位置 | 标志性表达 | 示例 |
|------|-----------|------|
| 起始 | "可能构成"、"或许符合" | "甲的行为可能构成故意杀人罪" |
| 过程 | "为此需要"、"前提是" | "为此需要满足以下要件" |
| 归入 | "此处"、"本案中" | "此处，甲使用了足以致命的手段" |
| 结论 | "因此"、"综上" | "因此，甲的行为符合该要件" |

#### 1.1.2 适用场景

- 案件事实是否符合某一要件**存在争议**或**需要论证**
- 法律概念的适用范围不明确
- 需要运用法律解释方法
- 涉及学说争议

#### 1.1.3 完整示例

```
【总起句】甲将安眠药混入乙的饮料中的行为，可能符合刑法第232条
故意杀人罪中"杀害他人"的构成要件行为要素。

【定义】所谓"杀害他人"，是指以任何足以引发死亡结果的方式，
对他人生命法益实施侵害的行为。

【归入】此处，甲将超过致死剂量的安眠药（10倍常规剂量）混入乙的
饮料中。医学上，该剂量的安眠药足以抑制中枢神经系统功能，导致
呼吸衰竭而死亡。甲的行为在客观上属于以投毒方式对他人生命法益
实施侵害，具有引发死亡结果的高度可能性。

【结论】因此，甲的行为符合"杀害他人"这一构成要件行为要素。
```

### 1.2 裁判式

#### 1.2.1 核心特征

裁判式是以**结论为起点**、直接给出判断的论证模式。其本质是"先答后（简要）释"。

**逻辑结构**：

```
结论 → [简要理由]
```

**语言标志**：

| 位置 | 标志性表达 | 示例 |
|------|-----------|------|
| 起始 | "甲（不）符合"、"甲是/不是" | "甲是适格的行为主体" |
| 理由 | "因为"、"鉴于" | "因为甲系年满18周岁的自然人" |

#### 1.2.2 适用场景

- 要件的成立/不成立**显而易见**，无需论证
- 案件事实直接、明确地满足（或不满足）法律规定
- 不存在任何合理争议

#### 1.2.3 完整示例

```
甲系年满22周岁、精神正常的自然人，具有完全刑事责任能力。
```

### 1.3 混合运用——"五花肉"原则

#### 1.3.1 原则内容

> **有争议的问题用鉴定式详细展开，无争议的问题用裁判式一笔带过。如同五花肉，肥瘦相间。**

#### 1.3.2 运用示例

```
一、构成要件符合性

（一）客观构成要件

1. 行为主体
甲系年满25周岁的自然人，具有一般犯罪主体资格。[裁判式——显然成立]

2. 构成要件行为
甲向乙的饮料中投放超量安眠药的行为，可能符合"杀害他人"的
行为要件……[鉴定式——需要论证]

3. 行为对象
乙系有生命的自然人，属于适格的行为对象。[裁判式——显然成立]

4. 构成要件结果
乙已死亡，构成要件结果已发生。[裁判式——显然成立]

5. 因果关系
此处需要检验甲的投毒行为与乙的死亡之间是否存在条件关系
……[鉴定式——需要论证]
```

#### 1.3.3 判断标准

| 使用裁判式的条件 | 使用鉴定式的条件 |
|----------------|----------------|
| 事实直接对应法条文字 | 需要法律解释 |
| 无任何争议空间 | 存在学说分歧 |
| 一般读者均能得出相同结论 | 结论可能出人意料 |
| 属于检验的"过渡性"要素 | 属于核心争点 |

## 二、术语纪律

### 2.1 当事人称谓规范

#### 2.1.1 基本规则

- 案件事实中给定的称谓（如甲、乙、丙）贯穿全文
- 不得中途更换称谓
- 不得使用模糊的代称（如"行为人"代替具体人名）

#### 2.1.2 称谓对照表

| 允许 | 禁止 | 原因 |
|------|------|------|
| "甲" | "该男子" | 模糊，可能有多个男子 |
| "乙" | "被害人" | 预设了犯罪成立的结论 |
| "丙" | "犯罪嫌疑人" | 预设了犯罪成立 |
| "甲某" | "被告人" | 属于诉讼程序用语 |

**例外**：在全案结论部分可使用"正犯"、"共犯"等标签，因为此时犯罪已经论证成立。

### 2.2 罪名与法条引用

#### 2.2.1 罪名精确性

- 必须使用**完整准确的罪名**
- 不得使用笼统分类（如"财产犯罪"、"暴力犯罪"）

| 正确 | 错误 |
|------|------|
| 故意杀人罪 | 杀人罪 |
| 盗窃罪 | 偷窃罪 |
| 故意伤害罪（致人重伤） | 重伤罪 |
| 抢劫罪（入户） | 入室抢劫 |

#### 2.2.2 法条引用格式

**完整引用格式**：

```
刑法第[条]条第[款]款第[项]项[前段/后段/但书]
```

**示例**：

```
刑法第263条第1项（入户抢劫）
刑法第20条第2款（防卫过当）
刑法第23条第1款前段（未遂犯的定义）
```

**区分规则**：

| 术语 | 含义 | 示例 |
|------|------|------|
| 条 | 基本单位 | 第232条 |
| 款 | 条下的自然段 | 第232条第1款 |
| 项 | 用数字编号的子条目 | 第263条第1项 |
| 前段/后段 | 同一款内的前半句/后半句 | 第20条第3款前段 |
| 但书 | "但是"引导的例外规定 | 第13条但书 |

### 2.3 犯罪论术语精确性

#### 2.3.1 故意/过失的区分

| 术语 | 内涵 | 使用场景 |
|------|------|----------|
| 直接故意（第一级） | 明知+积极追求 | 行为人的直接目的 |
| 必然性故意（第二级） | 明知必然发生+接受 | 附带后果的确定认知 |
| 或然性故意（间接故意） | 认识可能性+放任/容忍 | "也无所谓"的态度 |
| 有认识过失 | 认识到可能性+轻信能避免 | 与间接故意的区分重点 |
| 无认识过失 | 应当预见而未预见 | 注意义务违反 |

#### 2.3.2 犯罪形态术语

| 术语 | 适用场景 | 不得混用 |
|------|----------|----------|
| 既遂 | 构成要件完全实现 | ≠"犯罪完成" |
| 未遂 | 已着手未既遂 | ≠"未完成" |
| 中止 | 自愿放弃/阻止结果 | ≠"自首" |
| 预备 | 为实行犯罪准备工具/条件 | ≠"犯意表示" |

#### 2.3.3 参与形态术语

> 注：中国《刑法》第25-29条明文规定主犯、从犯、胁从犯与教唆犯，并未采用"直接正犯/间接正犯/共同正犯/帮助犯"五分法。下表中的精细参与形态为学理教义学工具，用于实质定性分析；最终归责与刑罚适用必须回到《刑法》总则规定的框架（如帮助犯通常归入第27条从犯，教唆犯依第29条处理）。

| 术语 | 定义 | 区分要点 |
|------|------|----------|
| 直接正犯 | 亲自实施构成要件行为 | 亲手实施 |
| 间接正犯 | 利用他人为工具 | 优势地位+利用缺陷 |
| 共同正犯 | 基于共同决意共同实施 | 功能性支配 |
| 教唆犯 | 引起他人犯罪决意 | 犯意由教唆而生 |
| 帮助犯 | 为正犯提供帮助 | 非核心角色 |

## 三、法条引用规则

### 3.1 MCP核验指令

> **强制要求**：所有引用的法条内容，必须通过MCP工具进行核验。未经核验的法条引用视为不合格。

### 3.2 首次引用格式

当某一法条在报告中**首次出现**时，必须完整引用条文原文并附核验标记：

```markdown
> **刑法第232条**【故意杀人罪】
> 故意杀人的，处死刑、无期徒刑或者十年以上有期徒刑；情节较轻的，
> 处三年以上十年以下有期徒刑。
> 
> `[✓ 已核验]` / `[⚠ 待核验]` / `[✗ 核验失败]`
```

### 3.3 后续引用格式

同一法条再次出现时，简写即可：

```
根据刑法第232条的规定……
```

### 3.4 核验状态标记

| 标记 | 含义 | 出现条件 |
|------|------|----------|
| `[✓ 已核验]` | 条文内容经MCP工具确认准确 | MCP返回匹配结果 |
| `[⚠ 待核验]` | 暂未能通过MCP核验 | MCP暂不可用 |
| `[✗ 核验失败]` | MCP返回不匹配结果 | 条文引用可能有误 |

### 3.5 司法解释的引用

```markdown
> **最高人民法院关于审理抢劫案件具体应用法律若干问题的解释**
> 第X条：……
> 
> `[✓ 已核验]` / `[⚠ 待核验]`
```

## 四、脚注规范

### 4.1 四种脚注来源标识

| 标识 | 含义 | 适用场景 |
|------|------|----------|
| `[评注]` | 来自法律评注类著作 | 引用刑法评注/教科书的学说观点 |
| `[案例]` | 来自司法判例 | 引用具体判决/指导性案例 |
| `[联网]` | 来自网络检索 | 通过WebSearch获取的信息 |
| `[KB:]` | 来自知识库 | 通过知识库MCP获取的内容 |

### 4.2 脚注使用规则

#### 4.2.1 必须使用脚注的情况

- 引用他人观点（学说立场）
- 引用具体案例
- 引用司法解释的具体内容
- 引用统计数据或实证材料
- 通过外部工具获取的任何信息

#### 4.2.2 不使用脚注的情况

- **自行进行的法律解释**：当分析师基于条文文义进行的解释论证，无需脚注
- **涵摄过程中的事实描述**：直接引用案件事实的部分
- **逻辑推理的中间步骤**：纯粹逻辑演绎

#### 4.2.3 脚注格式

```markdown
正文内容。[^1]

[^1]: [评注] 参见参考文献②，第 XXX 页（具体作者与书名见 LICENSE 末尾"参考文献"清单）。
[^2]: [案例] 最高人民法院指导案例第XX号。
[^3]: [联网] 来源URL。
[^4]: [KB:criminal-law-annotated] 具体条目引用。
```

### 4.3 禁止前向引用

**规则**：脚注只能引用在论证时点已经存在的材料，不得"前向引用"报告后文中的结论。

```
✗ 错误："如后文所述（见第三部分），甲构成正当防卫……"
✓ 正确：在当前位置完成论证，或引用已论证过的前文结论
```

## 五、学说争议展开格式

### 5.1 展开触发条件

当某一法律问题存在学说争议，且不同立场可能导致**不同结论**时，必须展开争议。展开深度分级与"结论一致则不问理由"规则详见 `methodology/proportionality.md §四`。

### 5.2 标准展开格式

```markdown
#### [争议问题名称]

**问题提出**：[清晰描述争议所在]

**学说分歧**：

| 学说 | 核心主张 | 主要论据 |
|------|----------|----------|
| 甲说 | …… | …… |
| 乙说 | …… | …… |
| 丙说 | …… | …… |

**分析与选择**：

甲说的优点在于……，但其缺陷在于……
乙说的优点在于……，但其缺陷在于……

**本文立场**：采纳[X]说。理由如下：
1. ……
2. ……
3. ……

**立场贯彻**：按照所采纳的[X]说，本案中……
```

### 5.3 立场一致性要求

> **一旦选择某一学说立场，必须在全文中贯彻始终，不得在不同位置采用不同立场。**

如果确需在不同问题上采用来自不同体系的立场，必须论证两者的兼容性。学派立场配置（user-config 驱动）详见 `workflow/0-cold-start-interview.md §1.5`。

### 5.4 争议旗帜标注规则

在以下情形中，必须在正文中插入争议旗帜标注（不强制使用学说分歧表格，至少需一句话标注）。

**触发条件**（满足任一即触发）：

1. 最高人民法院的指导案例/司法解释与学理通说存在明显分歧
2. 不同学者在同一问题上发表了互斥的立场
3. 德国/日本刑法教义学与我国司法实践之间在该问题上存在张力
4. 该问题的结论将直接影响罪与非罪、此罪与彼罪的认定
5. 案情中存在与该问题相关的"边缘情形"（如手段的非典型性）

**最简标注格式**（用于非核心争议）：

```
【教义学争议】此处涉及……问题，学理上存在……（通说）与……（少数说）
的分歧。本文采……说，理由如下：……
```

**完整标注格式**（用于核心争议，即 §5.2 标准展开格式）：

学说分歧表格 + 立场选择 + 反对说回应。

**不得省略争议标注的情形**（硬性规则）：

| 情形 | 必须标注的理由 |
|------|---------------|
| 被害人自杀/自伤作为加重结果 | 通说与严格因果关系论存在分歧 |
| 不作为的作为义务来源（特别是先行行为引发） | 保证人地位的学理基础存在多种学说 |
| 违法性认识错误的可避免性 | 可避免性标准的判断存在分歧 |
| 结果加重犯的直接性要件 | 直接性关联的宽严标准存在分歧 |
| 共犯的从属性程度 | 限制从属性说与最小从属性说等存在分歧 |
| 因果关系中介入因素的评价 | 介入因素的异常性/独立性判断标准存在分歧 |

## 七、常见错误清单

### 7.1 体系性错误

| 编号 | 错误类型 | 错误表现 | 正确做法 |
|------|----------|----------|----------|
| E01 | 阶层顺序颠倒 | 在检验构成要件之前讨论正当防卫 | 严格遵循三阶层顺序 |
| E02 | 要素"体系流浪" | 将违法性要素放在构成要件中讨论 | 每个要素归入其所属阶层 |
| E03 | 中止犯位置错误 | 在构成要件内讨论犯罪中止 | 中止是刑罚解除事由，置于三阶层之后 |
| E04 | 未遂与中止混淆 | 将中止等同于未遂 | 未遂是犯罪形态，中止是刑罚解除 |
| E05 | 跳过阶层 | 构成要件尚未确认就直接讨论有责性 | 必须依序通过每一阶层 |

### 7.2 概念混淆错误

| 编号 | 错误类型 | 错误表现 | 正确做法 |
|------|----------|----------|----------|
| E06 | 行为能力≠责任能力 | 将民法行为能力标准用于刑事责任 | 刑事责任能力有独立标准（刑法第17-19条） |
| E07 | 无罪推定≠存疑有利 | 在实体法论证中使用"无罪推定" | 实体法中使用"存疑时有利于被告" |
| E08 | 违法性认识≠构成要件认识 | 将对违法性的错误等同于事实认识错误 | 前者是禁止错误（违法性认识在中国法属学理处理），后者是构成要件错误（亦属学理；不得混引中国《刑法》第16条——该条是不可抗力） |
| E09 | 共同正犯≠共犯 | 将"共犯"仅指帮助犯/教唆犯 | 狭义共犯≠共同正犯，注意语境 |
| E10 | 因果关系≠客观归责 | 仅检验条件说即认为归责成立 | 因果关系(+)之后还需客观归责检验 |

### 7.3 论证方法错误

| 编号 | 错误类型 | 错误表现 | 正确做法 |
|------|----------|----------|----------|
| E11 | 先查因果后查结果 | 在确认结果发生之前讨论因果关系 | 必须先确认结果已发生，再检验因果 |
| E12 | 假设事实 | 在论证中添加案件未给出的事实 | 严格限于题目给定的事实 |
| E13 | 结论与标题矛盾 | 标题写"可能构成X罪"但结论认定不构成Y罪 | 结论必须回应标题中的假设 |
| E14 | 总起句缺失 | 直接开始定义和归入，无假设引导 | 每段涵摄必须以总起句开始 |
| E15 | 结论句遗漏 | 归入之后不给出明确结论 | 每段涵摄必须以结论句收尾 |

### 7.4 形式错误

| 编号 | 错误类型 | 错误表现 | 正确做法 |
|------|----------|----------|----------|
| E16 | 称谓不一致 | 时而称"甲"时而称"被告" | 全文统一使用案件给定称谓 |
| E17 | 罪名不精确 | 使用"杀人"代替"故意杀人" | 使用完整法定罪名 |
| E18 | 法条引用不精确 | 仅引用"刑法第XX条" | 精确到款、项、前段/后段 |
| E19 | 裁判式用于争议问题 | 对有争议的要素直接下结论 | 有争议的问题必须使用鉴定式 |
| E20 | 鉴定式用于显然问题 | 对显然成立的要素进行冗长论证 | 显然成立的问题使用裁判式 |

### 7.5 实体法错误

| 编号 | 错误类型 | 错误表现 | 正确做法 |
|------|----------|----------|----------|
| E21 | 正当防卫的时间要件错误 | 将事后防卫认定为正当防卫 | 侵害必须"正在进行" |
| E22 | 未遂的着手判断过早 | 将预备行为认定为着手 | 着手要求直接危及法益 |
| E23 | 共犯从属性忽视 | 在正犯行为不违法时认定共犯成立 | 学理通说采限制从属性：正犯至少构成要件+违法（中国《刑法》对此无明文） |
| E24 | 间接故意认定草率 | 仅因行为危险就认定间接故意 | 必须论证认识要素+意志要素 |
| E25 | 过失犯注意义务泛化 | 以一般生活经验代替具体注意义务标准 | 必须识别具体的注意义务来源 |

### 7.6 自查清单

在提交报告前，逐项检查：

```
□ 每个事实单元是否均已分析？
□ 每个参与人是否均已检验？
□ 每个可能的罪名是否均已涉及？
□ 三阶层顺序是否严格遵守？
□ 总起句-定义-归入-结论是否完整？
□ 鉴定式/裁判式的选择是否恰当？
□ 学说争议是否按格式展开？
□ 法条是否经过MCP核验？
□ 脚注标识是否正确？
□ 称谓是否全文一致？
□ 结论是否与标题对应？
□ 竞合处理是否完成？
□ 全案结论是否涵盖所有参与人？
□ 中止犯是否置于正确位置？
□ 无案件事实外的假设？
```
