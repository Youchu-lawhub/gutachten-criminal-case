# 结果加重犯

> **体系定位**：结果加重犯是一种特殊的犯罪类型，其结构为"基本犯罪 + 加重结果"。立法者对某些具有内在特别危险性的犯罪，规定当该犯罪导致更严重结果时加重处罚。

---

## 一、结果加重犯的概念与结构

### 1.1 定义

结果加重犯是指行为人实施基本犯罪，且该基本犯罪导致了法律规定的更严重结果，从而适用加重法定刑的犯罪类型。

### 1.2 本质

```
结果加重犯 = 基本犯罪（故意/过失）+ 加重结果 + 特殊因果关系 + 至少过失

结构特征：
- 不是简单的"基本犯 + 独立过失犯"的叠加
- 而是一种"一体化的犯罪类型"
- 加重结果须为基本犯罪"固有特别危险"的实现
```

### 1.3 典型罪名（中国刑法）

| 基本犯 | 加重结果 | 法律依据 |
|--------|----------|----------|
| 故意伤害 | 致人死亡 | 《刑法》第234条第2款 |
| 抢劫 | 致人重伤/死亡 | 《刑法》第263条 |
| 强奸 | 致使被害人重伤/死亡 | 《刑法》第236条 |
| 非法拘禁 | 致人重伤/死亡 | 《刑法》第238条 |
| 绑架 | 致使被绑架人死亡 | 《刑法》第239条 |
| 放火/爆炸等 | 致人重伤/死亡或重大财产损失 | 《刑法》第115条 |
| 交通肇事（逃逸） | 因逃逸致人死亡 | 《刑法》第133条 |

## 二、完整检视程式

```
前置检验：
  - 确认该罪名存在结果加重犯的法律规定
  - 确认加重结果已实际发生

I.   基本犯罪成立
     → 按对应犯罪类型的检视程式完整检验

II.  构成要件符合性（加重部分）
     A. 加重结果发生
     B. 基本犯罪行为与加重结果之间的因果关系
     C. 直接性要件
     D. 对加重结果的主观要件（至少过失）

III. 违法性
IV.  有责性
V.   其他要件
```

---

## III. 各要件详解

### I. 基本犯罪成立

- 须完整检验基本犯罪的全部成立要件
- 基本犯罪须至少达到未遂阶段（基本犯既遂非必要——争议见后文）
- 按对应的检视程式进行：
  - 故意作为犯 → `intentional-commission.md`
  - 过失犯 → `negligent-commission.md`
  - 不作为犯 → `omission.md`

**特别注意**：基本犯罪的成立是结果加重犯的前提；基本犯不成立，则结果加重犯无从谈起。

---

### II. 构成要件符合性（加重部分）

#### A. 加重结果发生

```
检验要点：
1. 加重结果为法定结果类型：
   - 死亡（"致人死亡"）
   - 重伤（"致人重伤"）
   - 重大财产损失
   - 其他法定加重结果
2. 结果已实际发生
3. 结果的程度达到法定标准
```

#### B. 因果关系

```
基本犯罪行为 → 加重结果

检验：
1. 条件关系：
   - "若无基本犯罪行为，加重结果是否不会发生？"
2. 注意：此处的因果关系检验与一般犯罪相同
3. 但加重结果须源于"基本犯罪行为"本身
   （而非与犯罪无关的偶然因素）
```

#### C. 直接性要件

> **核心要件**——结果加重犯区别于"基本犯 + 独立过失犯想象竞合"的关键

##### 概念

加重结果须为基本犯罪**固有的特别危险**的**直接实现**。

##### 判断标准

```
直接性 = 加重结果为基本犯罪类型所特有的危险性的实现

具体要求：
1. 危险的特殊性：
   - 加重结果须源于基本犯罪"固有的"/"典型的"危险
   - 非任何与犯罪时空关联的危险
   - 须为该犯罪类型"典型地附带"的危险

2. 实现的直接性：
   - 加重结果在因果流程上"直接"由基本犯罪行为导致
   - 无重大的因果中断
   - 非通过完全独立的因果流程实现

3. 排除情形：
   - 纯粹偶然的时空联系 → 不满足直接性
   - 被害人完全异常的自我行为 → 可能排除
   - 第三人的独立故意行为介入 → 可能排除
```

##### 典型案例分析

| 案例 | 直接性判断 | 理由 |
|------|-----------|------|
| 故意伤害 → 伤口感染 → 死亡 | **肯定** | 伤害行为固有的感染危险实现 |
| 故意伤害 → 送医途中车祸 → 死亡 | **否定** | 车祸非伤害行为的固有危险 |
| 抢劫 → 被害人逃跑摔下楼梯 → 死亡 | **肯定** | 被害人恐惧逃跑是抢劫的典型伴随危险 |
| 抢劫 → 被害人多年后因心理创伤自杀 | **否定** | 因果链过长，直接性中断 |
| 非法拘禁 → 被害人试图逃脱坠楼 → 死亡 | **肯定** | 试图逃脱是非法拘禁的典型伴随危险 |
| 强奸 → 被害人事后羞愤自杀 | **争议** | 是否为强奸固有危险的实现？倾向否定 |
| 放火 → 消防员救火中牺牲 | **肯定** | 引发救援是放火的典型伴随危险 |

##### 直接性要件的功能

```
功能分析：
┌─────────────────────────────────────────────────┐
│ 若无直接性要件：                                 │
│                                                  │
│ 基本犯 + 任何后续死亡 = 结果加重犯               │
│ → 处罚范围过宽，变相为"结果责任"                │
│                                                  │
├─────────────────────────────────────────────────┤
│ 有直接性要件：                                   │
│                                                  │
│ 基本犯 + 固有危险实现的死亡 = 结果加重犯         │
│ → 合理限缩处罚范围，体现罪责原则                │
└─────────────────────────────────────────────────┘
```

#### D. 对加重结果的主观要件

##### 主观要件的类型

```
结果加重犯的主观结构：

对基本犯罪：故意（通常情形）
对加重结果：至少过失

具体组合类型：
┌──────────────────────────────────────────┐
│ 类型一："故意 + 过失"型（最常见）        │
│ - 基本犯罪：故意                         │
│ - 加重结果：过失                         │
│ - 例：故意伤害致人死亡                   │
│       （故意伤害 + 过失致死）            │
├──────────────────────────────────────────┤
│ 类型二："故意 + 故意"型                  │
│ - 基本犯罪：故意                         │
│ - 加重结果：故意                         │
│ - 例：抢劫中故意杀人                     │
│ - 注意：此时需区分是否应独立评价为       │
│   故意杀人罪（竞合问题）                 │
├──────────────────────────────────────────┤
│ 类型三："过失 + 过失"型（较少）          │
│ - 基本犯罪：过失                         │
│ - 加重结果：过失                         │
│ - 例：过失致火灾 + 致人死亡             │
└──────────────────────────────────────────┘
```

##### 过失的判断标准

对加重结果的过失判断，适用 `negligent-commission.md` 中的标准：
- 客观预见可能性：理性一般人能否预见加重结果？
- 结果避免义务：是否应采取措施防止加重结果？
- 义务违反关联：遵守义务能否避免加重结果？

---

### III. 违法性

- 与基本犯的违法性检验合并
- 正当化事由须覆盖基本犯罪行为本身
- 若基本犯罪行为被正当化 → 结果加重犯整体不成立

### IV. 有责性

```
有责性检验：
1. 责任能力：覆盖行为时整体状态
2. 违法性认识：对基本犯罪的违法性认识
3. 期待可能性
4. 对加重结果的主观注意义务违反（如为过失型）：
   - 行为人个人能否预见加重结果？
   - 行为人个人有无避免加重结果的能力？
```

### V. 其他要件

与一般犯罪相同。

## 三、特殊问题

### 3.1 基本犯未遂 + 加重结果既遂

```
问题：基本犯仅为未遂，但加重结果已发生，如何处理？

例：甲意图重伤乙，一拳打中乙太阳穴，乙当场死亡。
    甲的重伤故意未实现（基本犯未遂），但死亡结果已发生。

处理方案：
通说：仍可成立结果加重犯
  - 理由：加重结果源于基本犯行为的固有危险
  - 基本犯行为已着手即产生了特有危险
  - 该危险在加重结果中实现 → 满足直接性要件
  
少数说：不成立结果加重犯
  - 理由：结果加重犯以基本犯既遂为前提

本程式库立场：采通说——基本犯至少着手即可。
```

### 3.2 结果加重犯的未遂

```
问题：是否存在"结果加重犯的未遂"？

情形一：基本犯既遂 + 加重结果未发生
  → 仅成立基本犯既遂，不存在结果加重犯的未遂
  → 理由：加重结果未发生，无从构成结果加重犯

情形二：基本犯未遂 + 加重结果未发生（行为人对加重结果有故意）
  → 争议
  → 肯定说：行为人故意追求加重结果 → 可成立结果加重犯的未遂
  → 否定说：结果加重犯以过失为基础，过失无未遂

本程式库立场：
  - "故意+过失"型：否定结果加重犯的未遂
  - "故意+故意"型：肯定结果加重犯的未遂可能性
```

### 3.3 结果加重犯的共同犯罪

```
问题：共同犯罪中，部分参与者对加重结果有过失，如何归责？

处理规则：
1. 各参与者须个别判断对加重结果的过失
2. 加重结果须源于各参与者"共同行为"的固有危险
3. 直接性要件须对每一参与者个别检验

具体分析：
- 共同正犯：
  → 若加重结果源于共同行为的固有危险
  → 且各共同正犯个别具有过失
  → 则各共同正犯均成立结果加重犯

- 教唆犯/帮助犯：
  → 若教唆/帮助行为使基本犯行为成为可能
  → 且共犯个人对加重结果有过失
  → 则共犯可能成立结果加重犯的教唆/帮助
  → 但须注意：直接性要件的限缩功能仍适用
```

### 3.4 与故意犯罪的竞合关系

```
问题：对加重结果为"故意"时，与独立故意犯罪的关系如何？

例：抢劫中故意杀人
  → 方案一：成立抢劫致人死亡（结果加重犯）—— 法条竞合（吸收）
  → 方案二：抢劫罪 + 故意杀人罪（想象竞合）
  → 方案三：结果加重犯包含故意致死，择一重罪处罚

中国法处理：
  - 取决于具体罪名规定：部分罪名的加重法定刑已涵盖故意致死（如抢劫致人死亡），此时按结果加重犯一罪处理
  - 若法条未将故意致死纳入加重构成，则成立基本犯 + 故意杀人罪的想象竞合，择一重罪处罚
  - 具体罪名有特别规定的从其规定
```

### 3.5 被害人特殊体质

```
问题：被害人因特殊体质导致加重结果发生

例：甲轻伤乙（一拳），但乙因血友病大出血死亡

分析步骤：
1. 因果关系：肯定（条件关系存在）
2. 直接性：伤害行为 → 出血 → 死亡 = 伤害固有危险的实现
3. 客观预见可能性：
   - 若行为人不知被害人特殊体质 → 是否属于一般人可预见？
   - 通说：特殊体质不排除客观可预见性（轻微伤害理论上都可能致死）
   - 但可在量刑中考虑

结论：通常仍肯定结果加重犯成立，但部分学说主张限缩（要求对特殊体质的认知）
```

## 五、与相关制度的区分

### 5.1 结果加重犯 vs 想象竞合

| 比较项 | 结果加重犯 | 想象竞合 |
|--------|-----------|----------|
| 本质 | 一个统一的犯罪类型 | 一行为触犯数罪名 |
| 结构 | 法定的"基本犯+加重结果" | 独立的数个犯罪构成 |
| 因果要求 | 直接性要件（特有危险实现） | 一般因果关系 |
| 处罚 | 适用加重法定刑 | 择一重罪处罚 |
| 法律效果 | 整体评价为一罪 | 形式上数罪但从一重处断 |

### 5.2 结果加重犯 vs 转化犯

| 比较项 | 结果加重犯 | 转化犯 |
|--------|-----------|--------|
| 结构 | 基本犯+加重结果=同一罪名的加重情节 | 基本犯+特定情节=转化为另一罪名 |
| 罪名 | 不变（仅法定刑加重） | 转变为新罪名 |
| 例子 | 故意伤害致死（仍为故意伤害罪） | 盗窃转化为抢劫（《刑法》第269条） |
