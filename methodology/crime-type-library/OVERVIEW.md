# 犯罪类型检视程式库：总览与使用指南

> 本检视程式库基于中华人民共和国现行《刑法》（2023 修正）。适用结论须以《刑法》明文规定为准（共同犯罪归入第 25–29 条主犯/从犯/胁从犯/教唆犯框架；未遂归入第 23 条等）。

## 一、程式库定位与目的

本程式库为犯罪构成检验引擎的**第三步骤——检视程式建构**提供标准化模板。

### 与引擎模块的对接关系

| 引擎模块 | 调用本程式库的时机 |
|----------|------------------|
| `../offense-examination-engine/03-schema-construction.md`（Step 3） | 先按 `06-general-framework.md` 选 9 类通用骨架 → 再依本程式库 7 类型文件叠加精化检视程式 |
| `../offense-examination-engine/04-element-decomposition.md`（Step 4） | 拆解最小涵摄单元时，对照本程式库相应文件中的要素清单 |
| `../offense-examination-engine/05-outline-assembly.md`（Step 5） | 大纲成型与争议标记时，参考本程式库标注的核心争议要素 |

**两库立场一致**：本程式库（细程式，7 类）是 `06-general-framework.md`（粗骨架，9 类）的精化版本，调用顺序为先粗后细，叠加而非替换。

### 核心功能

| 功能 | 说明 |
|------|------|
| 模板供给 | 为每一主要犯罪类型提供标准检视程式，作为构成要件检验的起始框架 |
| 体系保障 | 确保所有检验均遵循三阶层犯罪论体系的逻辑顺序 |
| 教义一致 | 在各犯罪类型间维持统一的理论立场，避免前后矛盾 |
| 适应基础 | 提供可依具体案件事实进行调整的基础结构 |

### 使用场景

本程式库服务于鉴定式案例研习的全过程：

1. **识别候选罪名**：引擎根据案件事实识别可能成立的犯罪
2. **匹配检视模板**：将候选罪名与本程式库中的对应类型模板匹配
3. **适应性调整**：根据具体构成要件条文和案件特点，对模板进行修改
4. **涵摄检验**：按照调整后的程式逐步进行涵摄

## 二、使用方法

### 2.1 模板选择流程

```
案件事实 → 识别候选罪名 → 判断犯罪类型
                                ↓
                    ┌─────────────────────────────┐
                    │ 主观面：故意 or 过失？        │
                    │ 行为面：作为 or 不作为？       │
                    │ 完成度：既遂/未遂/中止？       │
                    │ 参与面：单独犯 or 共同犯罪？    │
                    │ 特殊形态：结果加重？认识错误？  │
                    └─────────────────────────────┘
                                ↓
                    选择对应模板 → 适应性修改 → 检验
```

### 2.2 模板叠加规则

当案件涉及多种犯罪类型特征时，应按以下优先顺序叠加：

1. **基本模板**：以故意作为犯（intentional-commission.md）为基础
2. **行为方式修正**：如为不作为犯，叠加 omission.md 中的保证人地位等要素
3. **主观面修正**：如为过失犯，以 negligent-commission.md 替换主观构成要件部分
4. **参与形态修正**：如涉共同犯罪，叠加 joint-crime.md 中的对应参与类型
5. **特殊形态处理**：如涉结果加重，叠加 result-aggravated.md
6. **认识错误处理**：如涉错误，参照 error-theory.md 确定处理位置
7. **竞合处理**：最终阶段参照 concurrence.md 处理罪数问题

### 2.3 检验顺序原则

- **三阶层顺序不可逆转**：构成要件符合性 → 违法性 → 有责性
- **阶层内部顺序**：客观先于主观（构成要件层）；消极检验（违法性/有责性层）
- **多人案件**：先正犯后共犯
- **多罪案件**：先重罪后轻罪；同一行为人的各罪依次检验后统一处理竞合

## 三、犯罪类型分类体系

### 3.1 按主观面分类

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 故意犯 | 行为人认识并意欲（或容忍）构成要件事实的实现 | intentional-commission.md |
| 过失犯 | 行为人违反客观注意义务导致结果发生 | negligent-commission.md |

### 3.2 按行为方式分类

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 作为犯 | 以积极身体动作违反禁止规范 | intentional-commission.md / negligent-commission.md |
| 不作为犯 | 未为法律期待之行为，违反命令规范 | omission.md |

### 3.3 按完成程度分类

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 既遂犯 | 所有客观构成要件均已实现 | intentional-commission.md §A |
| 未遂犯 | 已着手但未完成全部构成要件 | intentional-commission.md §B |
| 中止犯 | 行为人自动放弃或有效防止结果 | intentional-commission.md §C |

### 3.4 按参与形态分类

> 注：中国《刑法》第25-29条明文区分主犯/从犯/胁从犯/教唆犯，本表展示的精细五分法（共同正犯/间接正犯/教唆犯/帮助犯/承继共同正犯等）为学理上的教义学工具，用于精细定性。

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 单独正犯 | 单独实施全部构成要件行为 | 各基本模板 |
| 共同正犯 | 基于共同犯罪计划共同实施 | joint-crime.md §B |
| 间接正犯 | 通过利用他人作为工具实施 | joint-crime.md §C |
| 教唆犯 | 引起他人犯罪决意 | joint-crime.md §D |
| 从犯（辅助作用，学理上称帮助犯） | 为他人犯罪提供物理或心理帮助 | joint-crime.md §E |

### 3.5 特殊犯罪形态

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 结果加重犯 | 基本犯+加重结果+因果关系+至少过失 | result-aggravated.md |
| 认识错误 | 事实错误/法律错误影响归责 | error-theory.md |

### 3.6 罪数与竞合

| 类型 | 说明 | 对应文件 |
|------|------|----------|
| 想象竞合 | 一行为触犯数罪名 | concurrence.md §4 |
| 法条竞合 | 数罪名间存在逻辑关系，仅适用一条 | concurrence.md §3 |
| 实质竞合 | 数行为成立数罪 | concurrence.md §5 |

## 四、本程式库采取的理论立场

### 4.1 犯罪论体系

- **三阶层犯罪论体系**：构成要件符合性 → 违法性 → 有责性
- **二元的不法论**：不法既包含行为无价值（行为方式本身的规范违反性），也包含结果无价值（行为对法益造成的实际侵害或危险）

### 4.2 构成要件层

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 因果关系 | 条件说+客观归责论 | 实现规范层面的限缩 |
| 故意认定 | 容任说为基准 | 通说地位，区分间接故意与有认识过失 |
| 打击错误 | 具体符合说为原则，标注法定符合说差异 | 学说争议显著，需双重展示 |
| 实行行为 | 混合主客观说 | 以行为人认知为基础，结合客观危险判断行为是否进入实行阶段 |

### 4.3 违法性层

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 正当化事由的主观要素 | 必要说 | 二元不法论的当然要求 |
| 假想防卫 | 限制法律效果的责任说 | 区别于严格责任说，兼顾行为人认知状态 |

### 4.4 有责性层

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 禁止错误 | 责任说：可避免仅减轻，不可避免排除责任（注：中国《刑法》对违法性认识无明文，本立场为学理通说） | 通说 |
| 期待可能性 | 承认其独立地位 | 极端情状下无法期待行为人守法时可超法规阻却责任（无明文依据，属学理通说） |

### 4.5 共同犯罪

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 正犯与共犯区分 | 犯罪事实支配理论/重要作用说 | 实质区分标准 |
| 共犯从属性 | 限制从属性说 | 共犯处罚以正犯行为具备构成要件符合性+违法性为足 |
| 共同正犯归责 | 部分实行全部责任（功能性犯罪事实支配） | 基于功能性犯罪事实支配 |
| 共犯过剩 | 仅就共同犯罪计划范围内归责 | 超出计划部分由实行者自负 |

### 4.6 过失犯

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 过失犯体系定位 | 二阶的过失犯论 | 客观注意义务违反定位于构成要件，主观注意义务违反定位于有责性 |
| 信赖原则 | 有条件承认 | 限缩注意义务范围 |

### 4.7 未遂犯

| 问题 | 采取立场 | 理由 |
|------|----------|------|
| 着手判断 | 混合主客观说（修正的客观说） | 以行为人想象为基础判断客观危险 |
| 中止自动性 | 主观说为基准 | "能达目的而不欲"vs"欲达目的而不能" |

## 六、使用注意事项

### 6.1 模板非机械适用

本程式库提供的是**标准检视程式**，而非不可变更的固定格式。使用时应注意：

- 具体罪名的构成要件可能要求增加或删减检视步骤
- 案件事实可能使某些检视步骤无争议而可简化
- 理论争议较大的问题应展示多种学说并明确表态

### 6.2 维护规则

- 理论立场如需变更，须同步更新所有受影响的检视程式文件
- 新增犯罪类型模板应遵循本程式库的统一格式与理论立场
