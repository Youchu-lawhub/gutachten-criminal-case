# Step 5：大纲成型

## 概述

大纲成型是检视程式引擎的最终输出步骤。在前四步完成了罪名发现、规范属性识别、检视程式构建和要件拆解之后，本步骤将所有成果组装为一份完整的、可直接指导鉴定式写作的案例分析大纲。大纲不仅是结构框架，更是包含事实映射、争议预测和写作密度指导的综合性文件。

---

## 一、大纲的本质定义

### 1.1 公式

```
大纲 = 检视程式 + 案件事实映射 + 争议预测 + 写作指导
```

### 1.2 四要素解析

| 要素 | 来源 | 功能 |
|------|------|------|
| 检视程式 | Step 3输出 | 提供逻辑骨架与检视顺序 |
| 案件事实映射 | 案件事实 × Step 4要件拆解 | 将事实材料对应至各要素节点 |
| 争议预测 | 学说分析 + 案件特殊性 | 标识需要深入论证的节点 |
| 写作指导 | 论证方法论 + 密度控制 | 指导各节点的论证深度与方法 |

### 1.3 大纲的角色

大纲是从"分析"到"写作"的桥梁。检视程式引擎的前四步属于分析阶段，大纲成型将分析结果转化为写作指令，使后续的鉴定式行文有据可依。

---

## 二、组装规则

### 2.1 外壳结构

大纲的最外层结构遵循以下层级：

```
案件分析大纲
├── 参与人A
│   ├── 罪名1（针对事实单元U1）
│   │   ├── 第一阶层各要素
│   │   ├── 第二阶层
│   │   └── 第三阶层
│   ├── 罪名2（针对事实单元U1）
│   │   └── ...
│   └── 罪名3（针对事实单元U2）
│       └── ...
├── 参与人B
│   └── ...
└── 竞合处理
    └── ...
```

排列顺序原则：
- 参与人层：按在案件中的重要程度排列（主犯优先）
- 罪名层：按法益严重程度排列（人身→财产→社会→国家）
- 事实单元层：按时间顺序排列

### 2.2 信息密度控制

每一大纲条目应包含以下信息：

| 字段 | 内容 | 是否必填 |
|------|------|----------|
| 检视要素 | 要素名称 | 必填 |
| 对应事实 | 案件中的具体事实摘要 | 必填 |
| 预判结论 | 肯定/否定/争议 | 必填 |
| 争议标记 | 核心争议/次要争议/明显 | 必填 |
| 论证方法提示 | 需要使用的解释方法或学说 | 争议项必填 |
| 参考来源 | 条文/司法解释/学说/案例 | 选填 |

### 2.3 争议标记系统

| 标记 | 含义 | 写作要求 |
|------|------|----------|
| **[核心争议]** | 案件的核心法律问题，结论取决于此 | 完整展开正反论证，引用学说，明确表态 |
| **[次要争议]** | 存在不同观点但非案件核心 | 简要展示争议，给出立场，无需详细论证 |
| **[明显]** | 结论明显，无争议 | 一句话涵摄即可，甚至可合并处理 |
| **[解释争议]** | 不确定概念的解释存在分歧 | 展开解释方法论分析 |
| **[事实争议]** | 事实本身的认定存在争议 | 基于证据分析进行事实认定论证 |

## 三、四步组装流程

### Step 5.1：排序

**目标**：确定大纲中各罪名的检视顺序。

**排序规则**：

1. **参与人层排序**：
   - 直接正犯 → 间接正犯 → 共同正犯 → 教唆犯 → 帮助犯
   - 如主犯地位明确：主犯优先于从犯

2. **罪名层排序**：
   - 同一事实单元内：从重到轻
   - 不同事实单元间：按时间顺序
   - 法律拟制罪名：紧随基础罪名之后

3. **事实单元排序**：
   - 按时间先后
   - 持续行为作为一个单元处理

4. **竞合关系确认**：
   - 想象竞合：标注"从一重罪处断"
   - 实质竞合：各罪分别评价
   - 法条竞合：标注排斥关系

### Step 5.2：填充

**目标**：将案件事实映射至检视程式的各要素节点，并标注争议等级。

**操作流程**：

1. 对每一构成要件要素：
   - 从案件事实中提取对应事实片段
   - 进行初步涵摄判断（肯定/否定/争议）
   - 标注争议等级

2. 对每一争议要素：
   - 识别争议性质（学说争议/解释争议/事实争议）
   - 列出主要观点及支持者
   - 预判最可能的结论
   - 标注需要使用的论证方法

3. 对正当化/免责事由：
   - 确认是否有事实基础
   - 有基础的：逐一填充子要素
   - 无基础的：标注"无需检视"

### Step 5.3：裁剪

**目标**：根据争议标记调整大纲详略，确保论证重心明确。

**裁剪规则**：

| 标记类型 | 裁剪处理 |
|----------|----------|
| [明显]要素 | 合并为一段概括性表述 |
| [明显]且连续出现 | 可合并为一组打包涵摄 |
| [核心争议] | 完整保留，增加论证路径提示 |
| [次要争议] | 保留要点，标注简要处理 |
| 无事实基础的正当化事由 | 删除或标注"不予检视" |
| 明显不成立的候选罪名 | 给出一句否定理由后排除 |

**展开规则**：

对于核心争议要素，展开以下子结构：
```
[核心争议] 要素X
├── 争议描述：……
├── 观点A（通说/判例立场）：……
│   └── 论据：……
├── 观点B（有力说/少数说）：……
│   └── 论据：……
├── 本文立场：……
│   └── 理由：……
└── 涵摄结论：……
```

### Step 5.4：交叉引用

**目标**：处理同一要素在多个罪名检视中重复出现的情况。

**操作规则**：

1. **识别重复要素**：
   - 同一行为事实可能在多个罪名中重复涵摄
   - 同一主观状态可能同时关涉多个罪名

2. **交叉引用标注**：
   - 首次出现：完整论证
   - 再次出现：标注"参见上文对[罪名A][要素X]的分析"
   - 需要补充论证时：标注"在上文基础上补充……"

3. **竞合处理引用**：
   - 在竞合部分引用各罪名的成立结论
   - 标注法条竞合时的排斥关系依据

## 四、两种输出模板

### 4.1 标准模板——表格式，用于用户确认

```markdown
## [参与人姓名] 的刑事责任分析

### 一、[罪名]（第XX条）——针对[事实单元描述]

| 检视层级 | 检视要素 | 对应事实 | 预判结论 | 争议等级 | 论证提示 |
|----------|----------|----------|----------|----------|----------|
| 一阶·客观 | 行为主体 | [事实] | 肯定 | [明显] | 简述 |
| 一阶·客观 | 实行行为 | [事实] | 争议 | [核心争议] | 展开A/B说 |
| 一阶·客观 | 结果 | [事实] | 肯定 | [明显] | 简述 |
| 一阶·客观 | 因果关系 | [事实] | 肯定 | [次要争议] | 条件说+合法替代 |
| 一阶·主观 | 故意 | [事实] | 肯定 | [明显] | 简述 |
| 二阶 | 正当防卫 | [事实] | 否定 | [核心争议] | 展开防卫限度 |
| 三阶 | 责任能力 | 无特殊情况 | 肯定 | [明显] | 一句话 |

**核心争议预览**：
1. [争议1]：[简要描述]
2. [争议2]：[简要描述]
```

### 4.2 简缩模板——线性式，用于内部驱动写作

```markdown
A的刑事责任

I. 故意杀人罪（《刑法》第232条）——对B的杀害行为
  1. 构成要件该当性
    a) 客观构成要件
      aa) 行为：A用刀刺B胸部 → [明显肯定]
      bb) 结果：B死亡 → [明显肯定]
      cc) 因果关系：刺伤→失血→死亡 → [明显肯定]
      dd) 客观归责 → [次要争议：被害人自我答责？]
    b) 主观构成要件
      aa) 故意：[核心争议：直接故意vs间接故意]
  2. 违法性
    a) 正当防卫（《刑法》第20条）→ [核心争议：防卫过当？]
      aa) 正在进行的不法侵害 → [明显肯定]
      bb) 防卫意思 → [明显肯定]
      cc) 必要性 → [核心争议]
      dd) 相当性/限度 → [核心争议]
  3. 有责性 → [明显肯定]
  4. 结论：A成立/不成立故意杀人罪（既遂/未遂）
```

## 五、向写作的过渡

### 5.1 大纲作为写作骨架

大纲直接转化为鉴定式写作的段落结构：
- 每一大纲条目 = 一个"大前提→小前提→涵摄→结论"的论证单元
- 争议标记决定论证密度
- 交叉引用避免重复

### 5.2 争议标记指导写作深度

| 争议标记 | 写作深度指导 |
|----------|-------------|
| [明显] | 一句话定义 + 一句话涵摄 + 结论。可合并多个明显要素。 |
| [次要争议] | 大前提（含争议展示）+ 立场表态 + 涵摄 + 结论。2-3段。 |
| [核心争议] | 完整大前提 + 正方论证 + 反方论证 + 本文立场及理由 + 涵摄 + 结论。4-6段。 |
| [解释争议] | 展开解释方法论（文义→体系→历史→目的）+ 结论。3-5段。 |

### 5.3 解释标记触发方法论工具包

当大纲中标注[解释争议]时，写作阶段应调用解释方法论工具包：

| 解释方法 | 适用场景 | 论证权重 |
|----------|----------|----------|
| 文义解释 | 确定语义边界 | 起点，但非决定性 |
| 体系解释 | 条文间协调 | 中等 |
| 历史解释 | 立法原意 | 参考性 |
| 目的解释 | 规范保护目的 | 决定性（通常） |
| 合宪解释 | 涉及基本权利 | 优先约束 |

## 六、八项质量标准清单

大纲成型后，须通过以下八项质量标准检验：

| 编号 | 标准 | 检查内容 | 通过标志 |
|------|------|----------|----------|
| 1 | **穷尽性** | 所有候选罪名是否均已纳入或明确排除 | 每一参与人×每一事实单元均有覆盖 |
| 2 | **逻辑性** | 检视顺序是否符合三阶层递进逻辑 | 无跳跃、无循环论证 |
| 3 | **准确性** | 要素定位是否正确，事实映射是否准确 | 每一要素对应正确的阶层与位置 |
| 4 | **重点性** | 核心争议是否突出，明显要素是否简化 | 争议标记分布合理 |
| 5 | **一致性** | 不同罪名检视中对同一事实的评价是否一致 | 无前后矛盾 |
| 6 | **经济性** | 是否避免不必要的重复论证 | 交叉引用正确使用 |
| 7 | **可写作性** | 每一节点是否包含足够信息以直接写作 | 事实、法律依据、论证方法均具备 |
| 8 | **可追溯性** | 每一判断是否可追溯至事实与法律依据 | 无悬空结论 |

### 质量检查操作

```yaml
quality_check:
  - criterion: "穷尽性"
    status: "通过/未通过"
    note: "[如未通过，说明遗漏项]"
  - criterion: "逻辑性"
    status: "通过/未通过"
    note: "[如未通过，说明逻辑问题]"
  # ... 依次检查全部8项
```

## 八、与写作阶段的衔接

大纲成型后，进入鉴定式写作阶段：

1. **逐节点展开**：按大纲顺序，对每一节点运用"大前提→定义→小前提→涵摄→结论"的鉴定式结构展开。
2. **争议深度控制**：严格按争议标记分配论证篇幅。
3. **交叉引用执行**：在重复要素处执行引用，避免冗余。
4. **结论汇总**：每一罪名检视结束给出明确结论；全文最后给出最终结论。

大纲是写作的充分准备——一份通过八项质量标准的大纲，应当使写作阶段只需关注语言表达与论证展开，而无需再作结构性思考。
