# Step 4：要件拆解

## 概述

要件拆解是将检视程式中的每一个构成要件要素进一步精细化分解的过程。其目的在于：将抽象的法律概念分解为可与案件事实直接对接的具体判断标准，为后续的涵摄奠定基础。

---

## 一、条文结构解析

### 1.1 "构成要件→法律效果"基本结构

每一刑法条文均遵循如下基本逻辑结构：

```
如果满足构成要件T，则产生法律效果R。
```

**示例：第264条（盗窃罪）**
- 构成要件T：盗窃公私财物，数额较大的，或者多次盗窃、入户盗窃、携带凶器盗窃、扒窃的
- 法律效果R：处三年以下有期徒刑、拘役或者管制，并处或者单处罚金

### 1.2 连接词识别

条文内部要素之间的逻辑连接关系：

| 连接词 | 逻辑关系 | 法律含义 | 示例 |
|--------|----------|----------|------|
| "……的" | AND（合取） | 所有要素必须同时满足 | "故意杀人的" |
| "或者" | OR（析取） | 择一满足即可 | "多次盗窃、入户盗窃、携带凶器盗窃、扒窃的" |
| "但是" / "但书" | EXCEPTION（例外） | 排除前文的适用 | "但是应当减轻或免除处罚" |
| "以……为目的" | ADDITIONAL（附加） | 在基本要素外增加特殊主观要素 | "以非法占有为目的" |
| "情节严重" / "数额较大" | THRESHOLD（门槛） | 行为需达到一定程度 | "诈骗公私财物，数额较大的" |
| "明知" | COGNITIVE（认知） | 要求行为人对特定事实有认识 | "明知是犯罪所得而予以窝藏" |

### 1.3 条文层次结构

```
条文
├── 第1款：基本构成要件 + 基本法定刑
├── 第2款：加重构成要件 / 加重法定刑
├── 第3款：特别加重 / 补充规定
└── 但书：例外/出罪条款
```

每一款、每一项均需独立进行要件拆解。

## 二、隐藏要素识别

### 2.1 条文未明示但必须检视的要素

许多构成要件要素并未在条文中明确表述，但根据犯罪论体系的要求，这些要素必须在检视中予以涵盖。

| 隐藏要素 | 说明 | 适用场景 |
|----------|------|----------|
| 因果关系 | 条文仅规定"致人死亡"，但因果关系需独立检视 | 所有结果犯 |
| 客观归责 | 制造法所不允许的风险并实现于结果中 | 因果关系确认后 |
| 故意 | 即便条文未明写"故意"，故意犯仍需检视故意 | 所有故意犯 |
| 过失 | 注意义务违反 + 预见可能性 | 过失犯 |
| 违法性认识可能性 | 通常默认存在，但特殊情况需独立检视 | 第三阶层 |
| 行为的社会相当性 | 通常不成问题，但在边界案件中需检视 | 日常行为构成犯罪时 |

### 2.2 识别方法

1. **与犯罪类型对照**：结果犯必有因果关系要素；故意犯必有故意要素。
2. **与三阶层体系对照**：每一阶层的核心要素均需检视，即使条文未提及。
3. **与争议热点对照**：学说争议往往集中在隐藏要素的认定上。

## 三、要件分类

### 3.1 按内容分类

| 分类 | 含义 | 典型要素 |
|------|------|----------|
| 主体要素 | 行为人应具备的身份/资格 | 国家工作人员、年满14周岁 |
| 行为要素 | 行为方式的描述 | 盗窃、杀人、诈骗 |
| 对象要素 | 行为作用的对象 | 公私财物、他人生命 |
| 主观要素 | 行为人内心状态 | 故意、过失、目的、动机 |
| 因果要素 | 行为与结果的因果联系 | 因果关系、客观归责 |
| 结果要素 | 行为造成的后果 | 死亡、伤害、财产损失 |
| 情节要素 | 行为的具体情节 | 数额较大、情节严重、多次 |

### 3.2 按确定性分类

| 分类 | 含义 | 拆解策略 |
|------|------|----------|
| 确定概念 | 含义明确，可直接适用 | 直接涵摄 |
| 不确定概念 | 含义需要解释才能适用 | 需展开解释方法论 |
| 一般条款 | 高度抽象，需大量价值填充 | 需类型化分析 + 类案比较 |

**确定概念示例**："年满14周岁"、"三年以上有期徒刑"
**不确定概念示例**："数额较大"、"情节严重"、"其他方法"
**一般条款示例**："情节显著轻微危害不大的"（第13条但书）

## 四、三种拆解方法

### 4.1 逐句拆解法

**适用场景**：条文较短、结构简单的罪名。

**操作步骤**：
1. 将条文逐句切分
2. 每一句中提取独立的构成要件要素
3. 标注要素间的逻辑关系
4. 对每一要素给出初步定义

**示例：第232条"故意杀人的"**
- 要素1：故意 → 认识到行为可能导致他人死亡 + 意志上希望或放任
- 要素2：杀人行为 → 足以导致他人死亡的行为
- 要素3：人 → 出生后、死亡前的自然人
- 隐含要素4：因果关系 → 行为导致死亡结果
- 隐含要素5：客观归责 → 制造并实现法所不允许的风险

### 4.2 层级展开法

**适用场景**：条文较长、要素复杂、存在多层嵌套关系的罪名。

**操作步骤**：
1. 识别条文的顶层结构
2. 对每一顶层要素进行下一层级的分解
3. 逐层展开直至每一末端要素可与事实直接对接

**示例：第263条抢劫罪（基本犯）**

```
抢劫罪构成要件
├── 客观构成要件
│   ├── 手段行为
│   │   ├── 暴力
│   │   │   ├── 对人实施
│   │   │   ├── 达到压制反抗程度
│   │   │   └── 当场实施
│   │   ├── 胁迫
│   │   │   ├── 以暴力相威胁
│   │   │   ├── 足以压制反抗
│   │   │   └── 当场实施
│   │   └── 其他方法
│   │       ├── 足以压制反抗
│   │       └── 当场实施
│   ├── 目的行为
│   │   ├── 劫取财物
│   │   └── 当场取得
│   └── 因果关系
│       └── 手段行为→压制反抗→取得财物
└── 主观构成要件
    ├── 故意
    │   ├── 对手段行为的认识与意志
    │   └── 对取财行为的认识与意志
    └── 非法占有目的
        ├── 排除权利人占有的意思
        └── 建立自己或第三人占有的意思
```

### 4.3 不确定概念标记法

**适用场景**：条文中包含大量不确定概念或需要价值判断的要素。

**操作步骤**：
1. 识别条文中所有不确定概念
2. 对每一不确定概念标记"需解释"
3. 预设可能的解释方案（文义/体系/目的/历史）
4. 标注司法解释是否已作具体化
5. 标注学说争议程度

**示例：第266条诈骗罪中的不确定概念标记**

| 要素 | 不确定程度 | 解释需求 | 已有具体化 |
|------|-----------|----------|-----------|
| 虚构事实 | 中 | 需界定"事实"范围（事实vs价值判断） | 部分司法解释 |
| 隐瞒真相 | 中 | 不作为欺诈的边界 | 学说争议中 |
| 数额较大 | 低（已量化） | 地区标准差异 | 司法解释已明确 |
| 错误认识 | 中 | 认识错误的具体内容与程度 | 学说分歧 |
| 处分行为 | 高 | 处分意识是否必要 | 重大争议 |

## 五、正当化事由/免责事由的定位

### 5.1 定位原则

对于每一正当化事由或免责事由，需明确其在检视程式中的功能定位：

| 功能定位 | 含义 | 判断标准 | 示例 |
|----------|------|----------|------|
| 否定构成要件要素 | 直接否定第一阶层某要素的成立 | 该事由使某构成要件要素不满足 | 被害人承诺否定"违反意志" |
| 独立正当化事由 | 在第二阶层独立阻却违法性 | 构成要件已该当，但行为具有合法性 | 正当防卫（第20条） |
| 独立免责事由 | 在第三阶层独立排除有责性 | 行为违法，但行为人不可归责 | 精神病免责（第18条） |

### 5.2 定位步骤

1. 确定案件中涉及的防御事由
2. 分析该事由的规范属性（Step 2已完成）
3. 判断该事由在本案中的具体功能：
   - 是否直接否定某构成要件要素？→ 第一阶层处理
   - 是否在构成要件该当后排除违法性？→ 第二阶层处理
   - 是否在违法性确认后排除有责性？→ 第三阶层处理
4. 在检视程式中正确安置

### 5.3 特殊情形：一事由跨阶层功能

某些防御事由可能在不同阶层发挥不同功能：
- **被害人承诺**：在财产犯罪中否定"违反意志"（第一阶层）；在人身犯罪中可能作为独立正当化事由（第二阶层）
- **义务冲突**：可能否定违法性（第二阶层），也可能影响期待可能性（第三阶层）

## 六、跨阶段双重挂载检查

### 6.1 问题描述

某些要素在犯罪论体系中可能在多个阶层发挥功能，需确认其在每一阶层的具体作用。

### 6.2 典型双重挂载情形

| 要素 | 第一阶层功能 | 第三阶层功能 | 说明 |
|------|-------------|-------------|------|
| 过失 | 构成要件层面的注意义务违反（客观归责） | 有责性层面的个人注意能力判断 | 过失犯采双重功能体系 |
| 故意 | 构成要件故意（对客观要素的认识与意志） | 罪责故意（可归责性的基础） | 主流说将故意同时定位于两阶层 |
| 违法性认识 | —— | 有责性层面的可归责性前提 | 仅在第三阶层检视 |
| 特殊身份 | 构成要件主体要素 | 可能影响量刑 | 主要在第一阶层，量刑阶段再次出现 |

### 6.3 操作规则

1. 在检视程式构建时标注双重挂载要素
2. 明确每一阶层检视该要素时的不同侧重：
   - 第一阶层：类型化判断（该行为是否属于X类型？）
   - 第三阶层：个别化判断（该行为人是否可归责？）
3. 在涵摄时注意区分两阶层的不同标准
4. 避免在第一阶层已否定的要素在第三阶层重复检视

## 六·一、分则解释 SOP 调用（基于实质解释论代表性方法论）

### 6.1.1 触发与产出契约

每个构成要件要素分解前，**必须**先做条款类型识别（注意规定/法律拟制/兜底/升格/空白/复行为/短缩二行为/共犯正犯化）；具体规则全文位于 `methodology/interpretation-toolkit.md` 下编，本节仅为引擎入口。

### 6.1.2 强制清单（agent 执行）

| 序号 | 操作 | 输出字段 | 详细规则位置 |
|---|---|---|---|
| 1 | 识别条款类型（简单/叙明/引证/空白罪状） | `clause_type` | interpretation-toolkit §十三.1 |
| 2 | 注意规定 vs 法律拟制鉴别（四要素） | `attention_or_fiction` | §十三.2 |
| 3 | 行为结构判定（单行为/复行为/短缩二行为） | `act_structure` | §十三.3 |
| 4 | 同位语 vs 独立要素三步测试 | `apposition_test_result` | §十三.4 |
| 5 | 升格条件分级（抽象/具体/行为方式） | `escalation_type` | §十三.5 |
| 6 | 兜底条款限缩三重过滤 | `catch_all_filter_passed` | §十四.2 |
| 7 | 用语相对性扫描 | `term_relativity_flags` | §十四.1 |
| 8 | 罪刑均衡反向校验 | `proportionality_check` | §十六 |

### 6.1.3 失败模式自检（W4 必走）

- ☐ 把同位语解释为独立行为（§133"发生重大事故"、§402多数"舞弊"、§224"虚构单位签合同"）；
- ☐ 把注意规定当法律拟制（致使不当限缩处罚范围）；
- ☐ 把法律拟制当注意规定（致使违反罪刑法定的扩张/类推）；
- ☐ 在法定实行行为前/后添加要素（如"为出卖而购买毒品"当贩卖既遂）；
- ☐ 把后行为视为前行为延伸（盗存折→骗取存款误判为盗窃罪整体）；
- ☐ 兜底条款无限缩；用此罪"情节严重"标准套彼罪；
- ☐ 抽象升格当作具体升格（误要求对加重结果有故意/过失）。

### 6.1.4 与 07-concept-discipline 的衔接

本节产出的 `clause_type` 与 `term_relativity_flags` 字段直接喂入 `07-concept-discipline.md`§四·一作为不确定法律概念锚定的输入。

## 八、与其他步骤的衔接

- **输入来源**：Step 3（检视程式构建）的完整检视程式树状结构。
- **输出去向**：Step 5（大纲成型）将要件拆解结果与案件事实映射，组装为完整的写作大纲。
- **迭代关系**：要件拆解中发现检视程式结构不当时，回溯Step 3修正；发现新的规范关联时，回溯Step 2补充。
