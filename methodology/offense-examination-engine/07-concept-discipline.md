# 不确定法律概念锚定纪律

> 本文件规定：当犯罪构成要件中出现不确定法律概念时，
> 必须遵循的定义锚定流程与一致性约束。

---

## 一、触发条件

当检视程式第四步（要件分解）中识别到的构成要件要素属于以下类型时，
自动触发本纪律：

### 1.1 典型不确定概念示例

| 类别 | 不确定概念 | 常见出处 |
|------|-----------|---------|
| 情节型 | "情节严重" | 《刑法》第226条、第236条、第266条等 |
| 情节型 | "情节特别严重" | 加重量刑档 |
| 数额型 | "数额较大" | 《刑法》第264条、第266条等 |
| 数额型 | "数额巨大" | 财产犯罪加重档 |
| 数额型 | "数额特别巨大" | 最高量刑档 |
| 手段型 | "特别残忍手段" | 《刑法》第232条、第234条等 |
| 对象型 | "公共安全" | 《刑法》第114-119条等 |
| 对象型 | "凶器" | 《刑法》第263条等 |
| 对象型 | "淫秽物品" | 《刑法》第364条等 |
| 状态型 | "公共场所" | 《刑法》第293条等 |
| 状态型 | "多次" | 《刑法》第264条等 |
| 主观型 | "明知" | 各罪名 |
| 主观型 | "以...为目的" | 目的犯 |

### 1.2 识别标准

符合以下任一条件即视为不确定法律概念：
- 条文使用价值评价性表述而非描述性表述
- 概念的外延须借助解释方法方能确定
- 同一概念在不同罪名中可能具有不同标准
- 概念适用须依赖司法解释或指导性案例确认具体数值/情形

## 二、四步解决路径

原则：**先到者优先** (first-to-arrive wins)。一旦在某步骤获得权威定义，
即停止继续检索，直接进入第四步写入。

### 步骤一：查阅本文件预设定义库

检查 `06-general-framework.md` 及本引擎内其他文件是否已对该概念提供明确定义。

- 若存在 → 直接引用，注明"内部预设定义"，进入步骤四
- 若不存在 → 进入步骤二

### 步骤二：经 MCP 工具检索权威定义

通过 MCP 法律数据库检索以下来源（按权威性递减排列）：

1. **立法释义 / 官方条文释义**
   — 全国人大法工委释义
2. **司法解释**
   — 两高（最高人民法院、最高人民检察院）司法解释
   — 两高联合发布的规范性文件
3. **指导性案例**
   — 最高人民法院指导性案例
   — 最高人民检察院指导性案例
4. **公报案例 / 典型案例**
   — 最高法公报案例
   — 两高发布的典型案例
5. **学理通说**
   — 权威教科书、评注书的多数观点

检索要求：
- 须明确标注来源文件名称、发布日期、文号
- 须确认来源文件现行有效（未被废止或修改）
- 若存在多个定义且相互矛盾 → 以效力层级高者为准；同层级以新法优先

结果处理：
- 若获得权威定义 → 进入步骤四
- 若未获得或来源可靠性不足 → 进入步骤三

### 步骤三：运用法律解释工具包自行建构工作定义

当步骤一、二均未获得满意定义时，须依以下解释方法层次建构工作定义：

```
(1) 文义解释
    — 概念的通常语义 + 法律语境中的可能文义边界
    → 确定文义射程

(2) 体系解释
    — 该概念在同一条文、同章、同编及其他法律中的用法
    — 与相邻概念的区分与关联

(3) 目的解释
    — 该条文所保护的法益
    — 立法目的对概念范围的指引

(4) 综合形成工作定义
    — 整合上述三层解释结果
    — 形成可操作的判断标准
    — 标注"工作定义，非权威来源"
```

注意：刑法严禁类推适用（罪刑法定原则 / nulla poena sine lege）。
工作定义不得超越文义射程的最大可能含义。

### 步骤四：写入涵摄并标注来源

将获得的定义写入鉴定式报告的涵摄环节：

格式要求：
```
[概念名称] 的含义：[定义内容]
  来源：[具体来源标注]
  来源类型：[立法释义 / 司法解释 / 指导性案例 / 工作定义]
  有效性：[现行有效 / 有效期至...]
```

## 三、一致性约束

### 3.1 同一概念同一定义

在同一份鉴定式报告中，同一不确定法律概念不得出现矛盾定义。

规则：
- 首次定义即为该报告中该概念的固定含义
- 后续出现同一概念时，须引用首次定义，不得另行解释
- 若发现前后矛盾，须回溯修正首次出现处

### 3.2 同一概念不同标准的处理

当同一文字表述在不同罪名中确有不同法定标准时（典型如"数额较大"在盗窃罪与
诈骗罪中的起点不同），须：

1. 明确标注所适用的具体罪名
2. 引用各自对应的司法解释条文
3. 在报告中以脚注或括号说明差异原因
4. 格式示例：
   ```
   "数额较大"：本罪（盗窃罪，《刑法》第264条）中指人民币1000-3000元以上
   （依据：两高2013年《关于办理盗窃刑事案件适用法律若干问题的解释》第1条）。
   注：此标准与诈骗罪中"数额较大"（3000-10000元以上）不同。
   ```

### 3.3 跨节/跨人一致性

在共同犯罪案件中，对同一构成要件要素的定义须在各参与人的检视中保持一致。

## 四、与犯罪类型检视程式的集成

### 4.1 自动触发机制

检视引擎第四步（要件分解）产出的每一个
构成要件要素，均须经过以下筛查：

```
for each 要素 in 构成要件要素列表:
    if 要素.类型 == "不确定法律概念":
        触发本纪律 → 执行四步解决路径
        将定义结果回写至要素定义字段
    else:
        继续常规涵摄
```

### 4.2 标记与追踪

- 所有经本纪律处理的概念须在报告中以特殊标记 `[已锚定]` 标注
- 未经本纪律处理但疑似不确定概念的，标记 `[待锚定]` 提示复核
- 最终报告提交前须确认无 `[待锚定]` 标记残留

## 四·一、用语相对性 / 同类解释 / 注意规定 vs 法律拟制（分则解释立场补充）

### 4.1.1 用语相对性原则

**核心命题**：**同一用语在不同罪名、甚至同一罪名的不同款项中，可以具有不同含义**（实质解释论代表立场，详见 `interpretation-toolkit.md` §十四.1）。

**判断标尺**：

1. 各罪名保护法益是否相同？（不同 → 用语含义可能不同）
2. 各罪名法定刑等级是否相同？（差异越大 → 含义差异越大）
3. 各罪名在分则体系中位置是否相同？

**经典分级**（高频对照）：

- **暴力**：抢劫罪（最广，含麻醉/灌酒/捆绑）→ 强奸罪（次之）→ 强制猥亵罪 → 妨害公务罪 → 招摇撞骗罪（不存在）；
- **占有**：盗窃罪（事实+规范的占有）→ 侵占罪（合法占有他人财物）→ 职务侵占/贪污（基于职务的占有）；
- **伪造**：货币伪造（含变造）→ 公文伪造 → 居民身份证伪造（仅指制作假证件）。

**详细规则**：见 `methodology/interpretation-toolkit.md` §十四.1。

### 4.1.2 同类解释（兜底条款限缩三步）

任何"其他XX""其他严重情节"条款，**必须**通过：① 同类性过滤；② 法益保护方向过滤；③ 处罚必要性过滤。详见 toolkit §十四.2。

### 4.1.3 注意规定 vs 法律拟制识别

任何引证罪状或与基本规定结果不一的条款，必须逐项过四要素：① 设立必要性；② 按基本规定解释结果；③ 能否推而广之；④ 文义射程是否需特别扩张。任何一项指向法律拟制即按拟制处理。详见 toolkit §十三.2。

### 4.1.4 普通用语规范化

"代为保管""遗忘物""贩卖""窝藏""非法债务"等用语必须以**规范意义 + 保护法益**重新校准；普通字面含义仅作底线（文义射程下限）。详见 toolkit §十二.1 文义解释 + §十八 代表性结论库。

## 五、常见错误与纠正表

| 序号 | 错误类型 | 错误表现 | 纠正方法 |
|------|---------|---------|---------|
| 1 | 跳过定义 | 直接涵摄而不先界定概念含义 | 回到本纪律步骤一重新执行；涵摄前必须有定义 |
| 2 | 循环同义定义 | 用概念本身或近义词解释概念（如"凶器即用于行凶的器具"） | 须指出概念的判断标准或构成要素，避免同义反复 |
| 3 | 过于宽泛的定义 | 定义未设边界，导致几乎任何情形均可涵摄 | 增加限定条件；参照司法解释中的列举 + 兜底条款结构 |
| 4 | 忽略制度语境 | 将A罪中的概念标准直接套用于B罪 | 确认概念在具体罪名中的特定含义；引用对应司法解释 |
| 5 | 虚假权威引用 | 捏造不存在的司法解释或案例编号 | 所有引用须经 MCP 工具验证；无法验证者标注"待核实" |
| 6 | 混淆故意/目的/明知 | 将"故意"等同于"目的"，或将"明知"等同于"间接故意" | 严格区分：故意=认识+意志；目的=超过的主观要素；明知=确定性认识 |
| 7 | 混同不同标准的"情节严重" | 将此罪"情节严重"的认定标准用于彼罪 | 逐罪检索对应司法解释中"情节严重"的具体情形列举 |
| 8 | 以学理观点替代规范依据 | 仅引用学术文献而忽略现行司法解释 | 学理观点仅在步骤三使用，且须标注；有司法解释时以司法解释为准 |

## 六、质量检查清单

在完成每个不确定概念的锚定后，须确认：

- [ ] 定义来源已标注且可追溯
- [ ] 定义未超越文义射程（罪刑法定 / nulla poena sine lege）
- [ ] 定义与本报告中其他位置对同一概念的使用一致
- [ ] 若为工作定义，已明确标注"非权威来源"
- [ ] 若同一概念在不同罪名有不同标准，已分别说明
- [ ] 所有引用的司法解释/案例编号已经 MCP 验证
- [ ] 未犯上述常见错误表中的任何一项
