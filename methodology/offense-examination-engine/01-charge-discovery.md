# Step 1：罪名发现（法益导向法）

## 概述

罪名发现是鉴定式案例研习的起点。在面对复杂案件事实时，分析者必须系统性地识别所有可能成立的罪名，避免遗漏。本步骤采用法益导向法，即从案件事实中受侵害或受威胁的法益出发，反向检索刑法分则中保护该法益的罪名条文，确保罪名发现的全面性与体系性。

---

## 一、法益导向法核心思路

### 1.1 基本逻辑

法益导向法的核心逻辑为：

```
案件事实 → 识别受侵害/受威胁的法益 → 定位刑法分则对应章节 → 筛选候选罪名
```

这一方法的优势在于：以法益为中介，将事实与规范连接，避免因经验不足导致的罪名遗漏。

### 1.2 四大法益类型与对应章节

#### （一）人身法益

保护对象包括：生命、健康、自由、人格尊严、名誉。

对应章节：刑法分则第四章「侵犯公民人身权利、民主权利罪」。

典型罪名群：故意杀人罪（第232条）、故意伤害罪（第234条）、过失致人死亡罪（第233条）、非法拘禁罪（第238条）、强奸罪（第236条）、侮辱罪/诽谤罪（第246条）等。

识别信号：事实中出现暴力、胁迫、限制人身自由、性侵害、侮辱性言行等。

#### （二）财产法益

保护对象包括：所有权、占有、经济利益、债权。

对应章节：刑法分则第五章「侵犯财产罪」。

典型罪名群：盗窃罪（第264条）、抢劫罪（第263条）、诈骗罪（第266条）、侵占罪（第270条）、敲诈勒索罪（第274条）、故意毁坏财物罪（第275条）等。

识别信号：事实中出现财物转移、欺骗获利、暴力取财、拒不返还等。

#### （三）社会法益

保护对象包括：公共安全、公共秩序、市场经济秩序、社会管理秩序。

对应章节：刑法分则第二章「危害公共安全罪」、第六章「妨害社会管理秩序罪」、第三章「破坏社会主义市场经济秩序罪」。

典型罪名群：放火罪（第114/115条）、危险驾驶罪（第133条之一）、生产销售伪劣产品罪（第140条）、妨害公务罪（第277条）等。

识别信号：事实中出现危及不特定多数人安全的行为、扰乱公共秩序、破坏市场规则等。

#### （四）国家法益

保护对象包括：国家安全、国家机关权威、司法秩序、国防利益。

对应章节：刑法分则第一章「危害国家安全罪」、第七章「危害国防利益罪」、第八章「贪污贿赂罪」、第九章「渎职罪」。

典型罪名群：间谍罪（第110条）、贪污罪（第382条）、受贿罪（第385条）、滥用职权罪（第397条）等。

识别信号：事实中出现国家工作人员、公共权力行使、国家秘密等要素。

## 二、构成要件行为锁定

### 2.1 从事实到行为

在法益定位完成后，需从案件事实中提取核心行为，将其与具体罪状进行匹配。

操作步骤：

1. **事实单元切分**：将整体事实分解为独立的行为单元，每一行为单元对应一个可能的犯罪评价。
2. **行为类型识别**：判断行为属于作为还是不作为。
3. **行为方式提取**：对照罪状描述中的行为方式要素（如"窃取"、"骗取"、"暴力"、"胁迫"）进行匹配。
4. **行为对象确认**：确认行为作用对象是否符合罪状要求。

### 2.2 注意要点

- 同一事实行为可能同时满足多个罪名的行为要素（想象竞合）。
- 注意区分行为的不同阶段（预备、着手、既遂、中止、未遂）。
- 不作为犯需特别注意保证人地位。

## 三、法律拟制条款并行检索

### 3.1 法律拟制的含义

法律拟制是指立法者明知两种情形本质不同，但出于刑事政策考量，规定将A情形"以B罪论处"。

### 3.2 典型法律拟制条款

| 条文 | 拟制内容 | 并行检索要求 |
|------|----------|--------------|
| 第269条 | 转化型抢劫（事后抢劫） | 在检视盗窃/诈骗/抢夺时，必须同时检视是否存在转化事由 |
| 第267条第2款 | 携带凶器抢夺以抢劫论 | 在检视抢夺罪时并行检索 |
| 第289条 | 聚众"打砸抢"以抢劫论 | 在检视故意毁坏财物等罪时并行检索 |
| 第196条第2款 | 恶意透支型信用卡诈骗 | 在涉及信用卡事实时并行检索 |
| 第382条第3款 | 受委托管理国有财产人员贪污以贪污罪论 | 涉及非国家工作人员管理国有财产时并行检索 |

### 3.3 并行检索规则

在发现候选罪名A时，必须检查：
- 是否存在以A罪论处的拟制条款（其他行为 → A）；
- A罪是否可能因拟制条款转化为B罪（A → 其他罪名）。

## 四、罪名穷尽检验

### 4.1 穷尽检验的基本要求

对于每一个事实单元和每一个参与人，必须系统性地检查所有可能成立的罪名。

### 4.2 检验维度

| 维度 | 检验内容 |
|------|----------|
| 既遂形态 | 犯罪是否已经完成 |
| 未遂形态 | 行为人是否已着手但未既遂 |
| 预备形态 | 是否存在可罚的预备行为（如第120条之一） |
| 中止形态 | 行为人是否自动中止或有效防止结果发生 |
| 正犯 | 直接正犯/间接正犯/共同正犯 |
| 共犯 | 教唆犯（《刑法》第29条）/帮助犯（对应《刑法》第27条从犯辅助作用） |
| 作为犯与不作为犯 | 是否存在不作为的替代评价可能 |

### 4.3 穷尽检验操作流程

```
对于每一参与人 P：
  对于每一事实单元 U：
    1. 列出该事实单元涉及的所有法益类型
    2. 对应每一法益类型，检索刑法分则所有可能匹配的罪名
    3. 对每一候选罪名，判断行为形态（既遂/未遂/预备/中止）
    4. 对每一候选罪名，判断参与形态（正犯/共犯）
    5. 检查法律拟制条款是否触发
    6. 记录所有通过初步筛选的候选罪名
```

## 五、MCP 法律检索协议

本步骤的罪名发现、条文定位、司法解释获取、案例佐证全部通过用户在冷启动访谈中配置的 `legal_search` 能力槽（参见 `workflow/0-cold-start-interview.md` §2.5）动态完成。文档中工具名一律以占位符形式书写：`${LEGAL_SEARCH.<能力名>}`，由 Agent 在调用前替换为 `.user-config.md` 中 `legal_search.tools.<能力名>` 字段的实际取值。能力槽要求所选服务商至少提供"已知条号取条文"和"语义检索"两项能力，且每条返回带可识别的时效字段（由 `legal_search.contract.timeliness_field` 指定，默认 `timeliness`）。

### 5.1 五项能力槽对照

| 能力（占位符） | 适用情景 | 入参约定 | 关键返回字段（按契约） |
|------|------|------|------|
| `${LEGAL_SEARCH.get_article}` | 已知罪名条号 → 取条文原文 | `title`（按 `${title_format}`，默认中文全称如"中华人民共和国刑法"）+ `number`（按 `${number_format}`，默认中文数字如"第二百六十四条"） | `article`(原文)·`${timeliness_field}`·`url` |
| `${LEGAL_SEARCH.search_article}` | 不知确切条号 / 找司法解释 / 数额标准 | `text`(关键词或自然语句) | 列表（约 5 条）：`title`·`article`·`doc_no`·`${timeliness_field}`·`${effectiveness_field}`·`issue_department`·`issue_date` |
| `${LEGAL_SEARCH.get_law_list}` | 找前置法（行政犯）整部法规 | `title` + `fulltext` 至少其一非空 | 法规列表前 10 条 |
| `${LEGAL_SEARCH.search_case}` | 找指导性 / 公报 / 参考案例 | `text` | 案例列表 |
| `${LEGAL_SEARCH.get_case_list}` | 案例标题 / 正文关键词粗筛 | `title` + `fulltext` | 案例列表前 10 条 |

### 5.2 罪名发现的标准检索决策树

```
案件事实 → 法益识别（§一）→ 章节定位（《刑法》分则十章）
              │
              ├─[情景 A] 罪名 + 条号已确定（如"264 盗窃"）
              │     → ${LEGAL_SEARCH.get_article}(title="中华人民共和国刑法", number="第二百六十四条")
              │     → 取 article 原文 + 校 ${timeliness_field}=="${timeliness_active_value}"
              │
              ├─[情景 B] 罪名已知，需司法解释 / 数额标准 / 构成要件细节
              │     → ${LEGAL_SEARCH.search_article}(text="<罪名> <要素关键词>")
              │       例："盗窃罪 数额较大 数额巨大"
              │     → 过滤 ${timeliness_field}=="${timeliness_active_value}"，按 ${effectiveness_field} 权威序排序
              │
              ├─[情景 C] 罪名未确定，仅有事实关键词
              │     → ${LEGAL_SEARCH.search_article}(text="<事实核心动词+对象>")
              │       例："伪造身份证 冒用"
              │     → 看返回标题中的罪名命中，再切到 [情景 A] 取条文
              │
              ├─[情景 D] 需要案例佐证（指导性 / 类案）
              │     → ${LEGAL_SEARCH.search_case}(text="<罪名> <争点>")
              │       例："正当防卫 防卫过当"
              │
              └─[情景 E] 行政犯 / 涉前置违法性（污染环境 / 非法经营 / 妨害传染病防治等）
                    → ${LEGAL_SEARCH.get_law_list}(title="反洗钱法") 等先确定前置法
                    → 再回情景 A/B 检验刑法条文
```

### 5.3 条号入参格式（get_article 必用）

`number` 入参格式由 `legal_search.contract.number_format` 决定。默认契约为"中文数字"，复杂条号格式：

| 阿拉伯 | 中文 | 阿拉伯 | 中文 |
|---|---|---|---|
| 第264条 | 第二百六十四条 | 第133条之一 | 第一百三十三条之一 |
| 第287条之二 | 第二百八十七条之二 | 第219条之一 | 第二百一十九条之一 |
| 第201条 | 第二百零一条 | 第20条 | 第二十条 |

若服务商返回阿拉伯数字或其他格式，agent 在冷启动 §2.5 自动探测时会将 `number_format` 写入配置，调用时按配置的契约转换。

### 5.4 返回结果四级过滤（铁律）

每次 `${LEGAL_SEARCH.search_article}` / `${LEGAL_SEARCH.search_case}` 返回列表后，按以下四级递进过滤：

1. **第一级 时效**：剔除所有 `${timeliness_field}` 不等于 `${timeliness_active_value}` 的条目，**不得引用**
2. **第二级 效力**：在 `${effectiveness_field}` 字段中按权威序排：`司法解释` ＞ `司法解释性质文件` ＞ `地方司法文件`
3. **第三级 主体**：`issue_department` 含"最高人民法院"/"最高人民检察院"为权威；地方高院为地方标准（注意江浙沪取数额上限、西部取下限）
4. **第四级 时序**：相同效力级别下取 `issue_date` 最新者作为引用首选

### 5.5 W2 阶段调用记录义务

每次 MCP 调用必须在 `verification-log.md` 中留痕，字段为：
- `tool`（实际解析后的工具名）
- `arguments`（实际入参）
- `${timeliness_field}`（返回时效状态）
- `gid` / `doc_no`（结果唯一标识）
- `url`（结果链接，便于人工复核）

### 5.6 与 criminal-law-annotated 的分工

| 资源 | 提供 | 调用时机 |
|------|------|---------|
| 法律检索 MCP（能力槽） | **现行法条 + 司法解释 + 案例**（动态、带时效） | 罪名发现、条文定位、案例佐证、终稿核验 |
| `criminal-law-annotated` skill（注释刑法全书知识库） | **立法理由 + 条文说明 + 学理评注**（静态切片） | 需要学理深度或立法背景时 |

二者互补：法律检索 MCP 校时效 → 进入正文论证；annotated 提学理 → 进入脚注评注层。

## 六、常见遗漏陷阱表

| 陷阱类型 | 具体表现 | 防范措施 |
|----------|----------|----------|
| 遗漏想象竞合罪名 | 只关注最终评价罪名，遗漏其他同时成立的罪名 | 对每一法益独立检索罪名 |
| 忽视法律拟制条款 | 未检查转化犯规定 | 对每一基础罪名检查是否存在拟制/转化条款 |
| 遗漏转化型犯罪 | 盗窃后使用暴力未检视第269条 | 建立拟制条款检查表 |
| 忽视预备/未遂形态 | 仅关注既遂，遗漏可罚的预备或未遂 | 对每一罪名逐一判断行为达到何种阶段 |
| 忽视不作为犯替代评价 | 仅以作为犯思路分析 | 对每一参与人检查是否存在保证人地位 |
| 遗漏身份犯 | 未注意行为人特殊身份可能触发的罪名 | 首先确认所有参与人的身份属性 |
| 遗漏结合犯/加重犯 | 仅检视基本犯，未注意加重构成 | 对每一基本罪名检查是否存在加重/结合构成 |
| 忽视共犯独立罪名 | 帮助行为本身可能独立构成犯罪 | 对帮助行为独立进行罪名检索 |

## 七、输出格式

### 7.1 候选罪名清单结构

```yaml
charge_discovery_output:
  case_id: "[案件编号]"
  participants:
    - name: "[参与人姓名]"
      role_in_facts: "[事实中的角色描述]"
      identity_attributes: "[身份属性，如国家工作人员等]"
      factual_units:
        - unit_id: "U1"
          unit_description: "[事实单元概要]"
          candidate_charges:
            - provision: "第XXX条第X款"
              charge_name: "[罪名]"
              protected_rechtsgut: "[保护法益]"
              completion_stage: "既遂/未遂/预备/中止"
              participation_form: "直接正犯/间接正犯/共同正犯/教唆犯/从犯（含辅助作用即学理上的帮助犯）"
              fiction_clause: "无/第269条转化"
              preliminary_assessment: "高度可能/需深入检视/可能性较低"
              notes: "[初步分析备注]"
```

### 7.2 输出要求

1. **穷尽性**：所有通过初步筛选的罪名均应列入，不得预先排除。
2. **层次性**：按法益类型分组排列。
3. **初步评估**：对每一候选罪名给出初步可能性判断，但不作最终结论。
4. **标注争议**：对存在争议的罪名认定标注争议点。
5. **检索记录**：记录MCP检索的查询语句与结果摘要。

## 八、与下一步骤的衔接

罪名发现完成后，候选罪名清单将进入：
- **Step 2（规范属性识别）**：对每一候选罪名条文进行规范属性分类。
- **Step 3（检视程式构建）**：为每一需要深入检视的罪名构建完整的三阶层检视程式。

罪名发现阶段的核心原则是「宁多勿漏」——在后续检视中排除不成立的罪名，远比事后发现遗漏更为妥当。
