# W3 附录：并行写作子代理提示词模板

> 本文件定义 W3 并行写作子代理（subagent）的提示词构造规范。
> 核心设计原则：**提示词仅包含结构性指令与文件路径，绝不内联案情事实描述**，
> 以规避刑事案例中敏感内容（强奸、贿赂、暴力等）触发模型内容安全过滤器。

---

## 一、铁律：零内联敏感内容

### 1.1 规则表述

> **Prompt 只含结构指令 + 文件路径，不含任何案情事实的实质性描述。**

### 1.2 禁止与允许

| 禁止 | 允许 |
|------|------|
| 在 prompt 中复述案情经过 | 引用文件路径 `[工作目录]/case-summary.md` |
| 描述具体犯罪行为细节 | 列明罪名名称 + 法条条号（如"盗窃罪 §264"） |
| 引用被害人/嫌疑人个人信息 | 使用大纲中的参与人别名（甲/乙/A） |
| 嵌入暴力、性侵、贪腐等场景文字 | 指向大纲文件让子代理自行读取 |

### 1.3 设计理由

- 内容安全过滤器对 prompt 文本做全文扫描，内联敏感描述易触发拦截
- 文件引用模式下，敏感内容仅在子代理内部读取，prompt 本身干净
- 罪名名称 + 条号属于法律术语，不触发安全策略

---

## 二、标准提示词模板

首次调用子代理时，使用以下模板：

```markdown
请阅读以下文件并完成鉴定式写作任务：

**必读文件**：
- 案情摘要：[工作目录]/case-summary.md
- 检视大纲：[工作目录]/outline.md 中关于 [参与人] 的部分
- 写作规则：methodology/gutachten-writing-rules.md
- 深度配比：methodology/proportionality.md
- 格式约束：workflow/w3-format-constraints.md

**写作范围**：
- 参与人：[姓名/别名]
- 罪名清单：[从大纲中提取的罪名列表，仅列罪名名+条号，不描述案情]

**写作要求**：
- 严格按照大纲结构展开，不得增删章节
- 使用鉴定式四步法（文义起点→体系协调→历史/目的支撑→结论）
- 核心争议要素（标记 **争议**）≥500字，非争议要素100-200字
- 每个罪名须包含三阶层完整检视 + 小结
- 法条引用使用 blockquote 格式 + 校验标签
- 格式硬性约束见 w3-format-constraints.md，违反任何一条需返工
```

### 模板变量说明

| 变量 | 来源 | 填充示例 |
|------|------|----------|
| `[工作目录]` | 当前会话工作路径 | `/path/to/workspace` |
| `[参与人]` | W1 参与人表中的别名 | `甲` |
| `[罪名列表]` | 闸二大纲中该参与人的候选罪名 | `盗窃罪 §264、抢劫罪 §263` |

## 三、降级提示词模板

当标准模板触发内容安全过滤时，切换至以下精简模板：

```markdown
[内容安全重试模式]
请阅读以下文件，按大纲结构完成写作：
- 案情：[工作目录]/case-summary.md（请自行读取）
- 大纲：[工作目录]/outline.md §[参与人]（请自行读取）
- 方法论：请自行读取 methodology/gutachten-writing-rules.md 和 methodology/proportionality.md

重要：不要在你的回复中复述案情事实细节，仅输出鉴定式论证正文。
```

### 降级模板的设计差异

| 维度 | 标准模板 | 降级模板 |
|------|----------|----------|
| 指令详细度 | 完整写作要求清单 | 最小化指令，依赖文件自身规范 |
| 文件引用方式 | 分类列出每个文件 | 合并引用，减少 prompt 文本量 |
| 安全提示 | 无（默认安全） | 显式要求子代理不复述事实 |
| 格式约束引用 | 直接列出 w3-format-constraints.md | 省略（子代理从方法论文件继承） |

## 四、重试协议

### 4.1 触发条件与处理流程

```
标准模板调用子代理
  → 成功返回完整文本 → 正常流转
  → 返回"检测到不合规内容"或空响应 → 自动切换降级模板
      → 降级模板调用成功 → 正常流转，标记 [降级模式完成]
      → 降级模板也失败 → 标记为 ✗ 阻断，请求用户介入
```

### 4.2 重试规则

| 序号 | 规则 | 说明 |
|------|------|------|
| R1 | 首次失败自动降级 | 标准模板 → 降级模板，无需用户确认 |
| R2 | 禁止原样重发 | 绝不将触发过滤器的同一 prompt 再次发送 |
| R3 | 降级仅一次 | 降级模板失败后不再重试，直接阻断 |
| R4 | 阻断时通知用户 | 输出阻断报告：触发阶段、子代理 ID、错误类型 |

### 4.3 阻断报告格式

```markdown
---
## 写作阻断报告

| 项目 | 内容 |
|------|------|
| 写作单元 | WU-[编号] |
| 参与人 | [别名] |
| 阻断阶段 | 标准模板 / 降级模板 |
| 错误类型 | 内容安全过滤 / 空响应 / 其他 |
| 处理建议 | 用户手动审阅案情后重新触发 / 调整写作范围 |
---
```

## 五、与主流程的集成

### 调用入口

- `3-drafting.md` §二（并行触发条件）中，每个写作单元启动时按本文件构造子代理 prompt
- 子代理接收 prompt 后，自行读取所列文件，完成写作并返回文本

### 状态回写

| 结果 | 回写动作 |
|------|----------|
| 标准模板成功 | 写作进度表标记 `✓ 完成`，无额外标注 |
| 降级模板成功 | 写作进度表标记 `✓ 完成`，附注 `[降级模式]` |
| 两次均失败 | 写作进度表标记 `✗ 阻断`，生成阻断报告 |

### 流转至下游

- 所有写作单元完成后 → 按 `3-drafting.md` §十一 流转至 W4
- 存在阻断单元 → 等待用户介入后继续
