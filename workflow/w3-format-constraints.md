# W3 并行写作格式硬约束

> 本文件定义 W3 并行写作阶段的二进制格式规则（必须/禁止），
> 确保多个 subagent 并行输出不同章节时保持格式一致性。
> 所有规则均为硬约束，违反即判定输出不合格。

---

## 一、必须（MUST）

以下 10 项规则为强制执行，每项写作单元输出前须逐项自检。

### M1 三阶层标题使用阿拉伯数字

三阶层犯罪论标题统一使用阿拉伯数字编号：

```
1. 构成要件符合性
2. 违法性
3. 有责性
4. 小结
```

每个罪名检视的三阶层 + 小结均以阿拉伯数字作为一级编号，不得更换编号体系。

### M2 小结标题统一

每个罪名检视的收尾节标题一律使用"小结"二字，格式为：

```markdown
4. 小结
```

小结内容格式为：`✓` 或 `✗` + 一句话理由。示例：

```
✓ 甲的行为符合《刑法》第264条盗窃罪的全部构成要件，且无违法性或有责性层面的阻却事由。
```

### M3 法条引用位置

法条引用（blockquote 格式）必须放置在"（二）正文"标题之后、"1. 构成要件符合性"标题之前。

结构示意：

```markdown
### [参与人] — [罪名] (§[条文])

（一）[前置内容，若有]

（二）正文

> 《中华人民共和国刑法》第XX条：
> "[条文全文]"
>
> [校验：日期，现行有效]

1. 构成要件符合性
   ...
```

### M4 交叉引用格式

当某一要素的论证已在其他参与人或罪名的检视中详述时，使用以下统一格式进行交叉引用：

```
已在[参与人姓名]§[罪名]§[条号]检视中详述
```

示例：

```
已在甲§盗窃罪§第264条检视中详述
```

不得自行发明其他交叉引用表述方式。

### M5 涵摄四步序列

每个要素的涵摄论证必须遵循以下四步序列：

```
文义起点 → 体系协调 → 历史/目的支撑 → 结论
```

各步骤含义：

| 步骤 | 内容 |
|------|------|
| 文义起点 | 法条或概念的文义范围界定 |
| 体系协调 | 与相关条文的体系关联与一致性分析 |
| 历史/目的支撑 | 立法背景与规范目的对解释方向的支撑 |
| 结论 | 该要素是否满足 + 简要理由 |

核心争议要素必须完整展开四步；非争议要素可简化为"定义 → 涵摄 → 结论"三步。

### M6 小结格式

小结统一使用以下格式：

```
✓/✗ + 一句话理由
```

- 使用 `✓` 表示成立、`✗` 表示不成立
- 理由限一句话，不展开论证
- 不重复前文已有的分析内容

### M7 核心争议要素字数下限

标记为 **争议** 的核心争议要素，论证展开不得少于 **500 字**。

- 字数计算范围：该要素标题下全部论证文本（含脚注引用，不含标题本身）
- 四步序列（文义起点 → 体系协调 → 历史/目的支撑 → 结论）每一步均需实质展开
- 学说分歧须列举至少两种立场并分析适用后果差异

### M8 非争议要素字数区间

未标记为 **争议** 的要素，论证展开控制在 **100-200 字** 之间。

- 简化为"定义 → 涵摄 → 结论"三步即可
- 不得过度展开，避免与核心争议要素的论证深度失衡
- 不得缩减至不足 100 字（仍须完成最低限度的涵摄论证）

### M9 脚注分类标注

所有脚注必须标注其来源类别，使用以下四种分类标签：

| 标签 | 含义 | 适用场景 |
|------|------|----------|
| `[评注]` | 刑法评注/教科书 | 引用学术评注或教科书观点 |
| `[释义]` | 法律释义 | 引用立法机关或权威机构的条文释义 |
| `[案例]` | 司法案例 | 引用指导案例、公报案例或裁判文书 |
| `[联网]` | 网络检索 | 通过 WebSearch 或 MCP 联网获取的信息 |

脚注示例：

```
实质解释论代表立场认为，盗窃罪中的"占有"应理解为事实上的支配与控制。[评注：参考文献②，p.1023]

最高人民法院指导案例第X号确认了该类行为的定性。[案例：指导案例X号]
```

### M10 术语一致性

同一概念在全文中必须使用同一术语表述，一旦选定某一用语指代特定概念，全篇不得更换。

- 以 W3 §3.3 术语表为基准
- 包括但不限于：构成要件符合性、违法性、有责性、涵摄、法益、正犯、共犯
- 若在写作过程中引入术语表中未列的新术语，须在各写作单元间保持一致

## 二、禁止（MUST NOT）

以下 6 项规则为绝对禁止，任何写作单元不得违反。

### P1 禁止替换"小结"标题

不得使用以下任何词语替代"小结"作为罪名检视收尾节标题：

| 禁止用语 | 说明 |
|----------|------|
| 中间结论 | 非标准标题 |
| 结语 | 非标准标题 |
| 总结 | 非标准标题 |
| 结论 | 非标准标题（小结本身即为结论，但标题必须用"小结"） |

唯一合法标题为"小结"。

### P2 禁止使用中文数字编号

三阶层标题不得使用中文数字编号：

| 禁止格式 | 正确格式 |
|----------|----------|
| 一、构成要件符合性 | 1. 构成要件符合性 |
| 二、违法性 | 2. 违法性 |
| 三、有责性 | 3. 有责性 |
| 四、小结 | 4. 小结 |

中文数字"一、二、三"仅用于外层大章节编号（如本文件中的"一、必须""二、禁止"），不得用于三阶层标题。

### P3 禁止法条前置

法条引用（blockquote 格式）不得放置在"（二）正文"标题之前。

- 法条引用的唯一合法位置：正文标题之后、构成要件符合性标题之前
- 案情概述或事实摘要部分不得嵌入法条全文引用

### P4 禁止自行增设章节

不得添加大纲中未列出的章节或节次。

- W3 严格按照闸二确认的检视大纲展开
- 若写作过程中发现需要新增检视维度，须标注 `[建议新增：XXX]` 并交由主 agent 裁决
- 不得未经确认即在输出中插入新章节

### P5 禁止合并罪名

不得将两个不同罪名的检视合并到同一节次中。

- 每个罪名必须独立成章，包含完整的三阶层 + 小结结构
- 即使两个罪名在事实上高度关联，也不得合并检视
- 罪名间的关联通过交叉引用（M4）和竞合处理解决

### P6 禁止非规范简称

不得使用非规范简称替代法律术语的完整表述。

| 禁止示例 | 正确表述 |
|----------|----------|
| "主观方面" 简写为 "主观" | 主观构成要件 |
| "客观方面" 简写为 "客观" | 客观构成要件 |
| "正当防卫" 简写为 "正当" 或 "防卫" | 正当防卫 |
| "紧急避险" 简写为 "避险" | 紧急避险 |
| "构成要件" 简写为 "要件" | 构成要件 |

法律术语的完整表述以术语表（W3 §3.3）和概念纪律文件为准。

## 三、违例处理

### 3.1 判定标准

违反任何一条"必须"（M1-M10）规则或触犯任何一条"禁止"（P1-P6）规则，该写作单元输出即判定为 **不合格**。

### 3.2 处理方式

| 情形 | 处理 |
|------|------|
| 单条规则违反 | 退回该写作单元，标注违例规则编号，要求定点修正 |
| 多条规则同时违反 | 退回该写作单元，列出全部违例项，要求整体返工 |
| 同一规则重复违反（经返工后仍违反） | 升级至主 agent 介入，人工审查格式问题根源 |

### 3.3 自检时机

- 每个写作单元输出完成后、提交合并前，由该 subagent 执行逐条自检
- 主 agent 在合并全部写作单元产出时，执行二次格式扫描
- W4 阶段 D3 维度格式扫描将再次核验本文件全部规则
