# W1：案情解析与设问识别

> 本文件定义工作流第一阶段：从原始案情材料中提取结构化事实要素，
> 识别设问，划分事实单元，为后续罪名检索 (W2) 提供规范化输入。

---

## 一、触发条件

- 用户配置已完成（`.user-config.md` 存在且有效）
- 用户提交案情材料（文本/文件/URL）

---

## 二、输入类型处理

### 2.1 纯文本输入

- 直接进入五节提取流程
- 无需额外解析步骤
- 保留原文全部内容，不做删减

### 2.2 文档输入

支持格式：`.docx`, `.pdf`, `.doc`

处理流程：
1. 检测文件格式
2. 调用解析工具：
   - 优先：MinerU 或其他文档解析 MCP
   - 备选：`docx` / `pdf` 技能
   - 兜底：`pdf-processing-anthropic` / `pdf-processing-openai`
3. 提取纯文本内容
4. 若解析失败 → 提示用户粘贴文本版本

### 2.3 网页 URL 输入

处理流程：
1. 使用 `WebFetch` 工具抓取页面内容
2. HTML → Markdown 转换
3. 提取正文部分（去除导航、广告等）
4. 若抓取失败 → 提示用户粘贴内容

### 2.4 混合输入

- 用户可同时提供多种输入
- 按提交顺序拼接，标注来源
- 冲突事实以用户明示为准

---

## 三、五节提取

**铁律**：五节提取为强制输出，总计不超过 70 行。逐节要求如下：

### §1 参与人表

| 别名 | 描述 | 法律地位 |
|------|------|----------|
| 甲/A | [简要特征描述] | 自然人 / 单位 |
| 乙/B | [简要特征描述] | 自然人 / 单位 |

规则：
- 使用案情中的原始称谓（甲乙丙丁 / 姓名 / 代号）
- 法律地位仅区分自然人与单位，不预判诉讼角色
- 标注年龄（若案情提及，涉及刑事责任能力判断）
- 标注职务/身份（若案情提及，涉及特殊主体犯罪判断）

### §2 时间线

格式：
```
[日期/时间段] | [描述性事件] | [关键行为节点标记]
```

规则：
- 日期精确到案情所提供的最小单位
- 事件描述使用中性语言，不含法律评价
- 关键陈述原文加引号保留：`"[原文]"`
- 关键行为节点标记：`★` 标识对构成要件判断有重大影响的时间点
- 时间不明确时标注 `[时间不详]`
- 因果链断裂处标注 `[因果断点]`

### §3 关键数字表

| 类别 | 数值 | 来源/语境 |
|------|------|-----------|
| 金额 | [具体数字] | [出处] |
| 数量 | [具体数字] | [出处] |
| 距离 | [具体数字] | [出处] |
| 时长 | [具体数字] | [出处] |
| 其他 | [具体数字] | [出处] |

规则：
- 保留原始单位
- 涉及数额犯门槛时标注 `[注意数额标准]`
- 无关键数字时本节输出"无关键数字"

### §4 设问逐字

规则：
- **逐字摘录**案情中明确的提问（如"试分析甲乙丙的刑事责任"）
- 若案情无明确设问 → 执行 AskUserQuestion：
  - question: "案情中未发现明确设问，请提供需要分析的问题："
  - header: "设问确认"
  - options:
    1. label: "分析全部参与人刑事责任" / description: "对所有参与人逐一进行完整鉴定"
    2. label: "仅分析特定参与人" / description: "指定需要分析的参与人范围"
    3. label: "针对特定罪名分析" / description: "仅就特定罪名进行检视"
- 设问决定后续分析范围，不可擅自扩大或缩小

### §5 事实单元划分

格式：
```
事实单元 [编号]：[中性描述标签]
  - 涉及参与人：[列表]
  - 核心行为：[简述]
  - 时间段：[起止]
```

规则：
- 以行为为划分标准，非以参与人为标准
- 使用中性描述标签（如"转账行为"而非"诈骗行为"）
- 正犯与共犯保持在同一事实单元内
- 同一行为的不同阶段（预备→实行→既遂）为同一单元
- 独立行为之间存在因果关联时标注 `[关联：单元X]`

### 三级切分标准

在上述基础规则之上，执行以下三级强制切分标准，防止"小行为被大行为吞并"：

**第一级（必须切分）——行为方式发生根本变化**：

同一参与人的不同行为方式必须切分为独立事实单元。

示例：生产假药 → 泄露内幕信息 → 向调查官员行贿 → 三者必须各自独立切分，不得合并为"甲的一系列违法行为"。

**第二级（必须切分）——行为对象发生变化**：

同一参与人对不同对象实施的行为必须切分为独立事实单元。

示例：对米兰的胁迫 → 对陈春的嘲讽 → 对陈大山的击打 → 三者必须各自独立切分。

**第三级（建议切分）——时空条件显著不同**：

同一参与人在不同时间段、不同场景下实施的同类型行为，若时空条件显著不同，建议切分。

示例：2007 年的高价采购 → 2007 年 12 月的特定合同 → 2014 年的清算伪造债务 → 三者应各自独立切分。

**特殊切分规则**：

- "以……相威胁/胁迫" + 威胁内容不同的 → 独立切分（每个威胁手段为一个独立单元）
- "酒后/醉酒" + 行为不同的 → 独立切分
- "被停职/被免职后" + 行为不同的 → 独立切分
- "得知……后/知情后" + 继续行为的 → 独立切分（知情前后为不同单元）

## 四、严格边界

### 本阶段禁止事项：

| 禁止内容 | 违规示例 | 违规后果 |
|----------|----------|----------|
| 罪名 | "甲构成盗窃罪" | 方法论违例 |
| 法条编号 | "根据第264条" | 方法论违例 |
| 争议预判 | "本案争议焦点在于..." | 方法论违例 |
| 法律评价 | "甲的行为具有违法性" | 方法论违例 |
| 结论性表述 | "甲应当承担刑事责任" | 方法论违例 |

### 方法论违例处理：
1. 立即删除违规内容
2. 回退至纯事实描述
3. 在输出末尾标注：`[已纠正方法论违例 × N处]`

---

## 五、复杂度层级

案情解析完成后，自动评估复杂度：

| 层级 | 条件 | 后续影响 |
|------|------|----------|
| 简单 | 1 参与人，1-2 事实单元 | W3 单线程写作 |
| 标准 | 2-3 参与人，2-4 事实单元 | W3 可并行 |
| 复杂 | 4+ 参与人，或多个交织事实单元 | W3 强制并行 + 协调协议 |

复杂度评估标准补充：
- 存在共犯关系 → 至少标准级
- 存在竞合问题 → 至少标准级
- 存在跨时间段连续行为 → 至少标准级

---

## 六、下游连接

### 输出物清单：
1. 五节提取结果（结构化文本）
2. 复杂度评估结果
3. 降级模式当前级别

### 流转至 W2：
- 输出直接作为 `2a-preselection.md`（闸一）的输入
- W2 基于事实单元逐一进行罪名检索

### 回环机制：
- W2 执行中发现事实遗漏 → 向 W1 发回补充请求
- 补充请求格式：
  ```
  [W2→W1 事实补充请求]
  缺失信息：[具体描述]
  涉及事实单元：[编号]
  补充方式建议：[询问用户 / 重新解析原文]
  ```
- W1 收到请求后：
  1. 优先从原始材料中补充提取
  2. 若原始材料无法满足 → AskUserQuestion 向用户确认
  3. 补充结果追加至对应节次，标注 `[W2请求补充]`

---

## 七、案例特征预扫描

**目的**：在五节提取完成后、进入 W2 之前，自动提取案例关键特征供下游参考，避免复杂特征到 W2/W3 才被发现导致返工。

**自动提取项**：

| 特征 | 提取方式 | 影响 |
|------|----------|------|
| 参与人数量 | §1 参与人表行数 | 决定 W3 并行度和复杂度层级 |
| 时间跨度 | §2 时间线最早→最晚 | 超过 5 年 → 提示刑法修正案时效适用问题 |
| 单位犯罪 | §1 法律地位含"单位" | 触发单位犯罪特殊主体检验 |
| 被害人特殊体质 | §2/§3 中的伤病/死亡描述 | 触发因果关系特殊判断（蛋壳脑袋等） |
| 跨法域要素 | 涉外/港澳台/境外关键词 | 提示管辖权和法律适用问题 |

**输出**：将提取结果附加在 case-summary.md 末尾的"案例特征概览"小节中，供 W2/W3 agent 参考。

## 九、情节节点检测

**目的**：在五节提取和案例特征预扫描完成后、进入 W2 之前，扫描时间线中的"转折型"情节节点，为 W2 罪名发现阶段提供预标记信号。

**与 W2 §九（情节节点检测）的分工**：
- 本步骤（W1）：仅在时间线和事实单元中**标记**转折型节点的存在，不涉及罪名
- W2 §九：基于此处标记的节点，触发具体的罪名升级检视清单

**检测项**（扫描 §2 时间线中的 `★` 标记节点）：

| 节点类型 | 触发关键词示例 | W1 标记方式 |
|----------|---------------|-------------|
| 知情状态转变 | "得知……后""知情后仍""获悉……继续" | 在时间线相应节点标注 `[情节节点：知情转变]` |
| 外部压力升级 | "投诉增多""调查介入""鉴定结论""监管通知" | 在时间线相应节点标注 `[情节节点：外部压力]` |
| 身份状态改变 | "被停职""被调离""被免职""被解聘" | 在时间线相应节点标注 `[情节节点：身份变更]` |
| 责任能力波动 | "酒后""醉酒""情绪激动""精神状态异常" | 在时间线相应节点标注 `[情节节点：能力波动]` |

**输出**：将检测结果附加在 case-summary.md 的"情节节点概览"小节中，供 W2 agent 调用 §九 时参照。

## 十、保存检查点

W1 完成且用户确认后，**必须**将五节提取结果 + 案例特征概览写入文件：

```
文件名：case-summary.md
保存位置：[工作目录]/case-summary.md
写入内容：五节提取结果（§1-§5）+ 复杂度评估 + 降级模式 + 案例特征概览（§七）
写入时机：用户确认 W1 提取结果后、流转至 W2 之前
```

此文件为 W4 D2 事实保真检查的基准数据源。未保存即进入 W2 视为流程违规。
