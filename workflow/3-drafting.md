## 二、写作单元划分

### 2.1 划分原则

- 一位参与人 = 一个写作单元
- 每个单元包含该参与人的全部候选罪名检视
- 单元内罪名按大纲顺序展开

### 2.2 动态负载均衡

写作单元划分后，预估每个单元的产出字数并据此调整分工：

**预估公式**：单罪名预估字数 = 核心争议要素数 × 600 + 非争议要素数 × 150

**分配规则**：

| 预估总字数 | 处理方式 |
|------------|----------|
| ≤ 15,000 字 | 正常分配为一个 subagent 任务 |
| 15,001–25,000 字 | 拆分为两个 subagent（按罪名组切分，如"财产/经济犯罪"与"人身犯罪"） |
| > 25,000 字 | 拆分为三个 subagent |

**高优先级先行**：标记为核心争议最多的单元优先启动，确保复杂单元有更多时间完成。

### 2.3 并行触发条件

| 参与人数 | 执行模式 |
|----------|----------|
| 1 人 | 单线程顺序写作 |
| ≥2 人 | 并行写作（每单元独立 subagent） |

### 2.4 单元输出产物

每个写作单元完成后产出：
- 该参与人完整鉴定文本
- 写作过程中新增的 MCP 校验记录
- 单元完成状态标记

## 三、并行写作协议

### 3.1 统一前提

所有并行写作单元共享以下前提：
- W1 事实认定结果（作为不可变输入）
- 闸一确认的候选罪名清单
- 闸二确认的检视大纲
- 降级模式级别

### 3.2 命名约定

```
参与人称谓：沿用 W1 §1 参与人表中的别名
事实单元编号：沿用 W1 §5 编号
罪名简称：[条文编号]-[罪名关键词]（如 264-盗窃）
```

### 3.3 术语表

并行写作前统一以下术语（按用户语言配置）：

| 中文术语 | 使用场景 |
|----------|----------|
| 构成要件符合性 | 第一阶层标题 |
| 违法性 | 第二阶层标题 |
| 有责性 | 第三阶层标题 |
| 涵摄 | 论证过程 |
| 法益 | 保护客体分析 |
| 正犯 | 参与形态 |
| 共犯 | 参与形态 |

### 3.4 Subagent 分工

每个 subagent 仅负责：
- 大纲中指定的章节范围
- 对应参与人的全部罪名检视
- 不得越界涉及其他参与人的结论

### 3.5 Subagent Prompt 安全构建（防内容过滤）

**铁律**：prompt 中**不得内联案情事实描述**（强奸、行贿、暴力等敏感题材会被平台安全过滤器误判）。prompt 只包含**结构指令 + 文件路径**，案情由 subagent 自行读取文件获取。

**标准模板**：见 `workflow/w3-prompt-template.md` §二。prompt 中仅写入：
- 必读文件路径（case-summary.md、outline.md、方法论文件）
- 参与人别名 + 罪名清单（仅罪名名 + 条号，不描述案情）
- 格式约束引用（w3-format-constraints.md）

**内容安全重试协议**：
1. subagent 首次返回"non-compliant content detected"或空响应 → 自动切换为 `w3-prompt-template.md` §三 的降级模板重试
2. 降级模板也失败 → 标记该单元为 `✗ 阻断`，请求用户介入
3. **禁止**将触发过滤的同一 prompt 原样重发

### 3.6 格式硬性约束

所有 subagent 产出必须遵守 `workflow/w3-format-constraints.md` 中列出的"必须/禁止"规则。违反任何一条即视为不合格产出，需返工。

## 四、实时 MCP 校验

### 4.1 校验触发条件

写作过程中，以下情形必须立即进行 MCP 校验：
1. 首次引用某一法条全文
2. 引用司法解释条文
3. 引用《刑法修正案》新增/修改条文
4. 阻却事由条款（闸二新增的条文）

### 4.2 校验流程

```
写作中引用法条 → 查询 verification-log.md
  → 已有记录且校验通过 → 直接引用，标注 [已校验]
  → 无记录 → 立即 MCP 查询 → 写入 verification-log.md → 引用
  → 已有记录但校验失败 → 不得引用，标注 [校验失败，已排除]
```

### 4.3 降级模式下的处理

- L0/L1：正常校验流程
- L2（MCP 不可用）：
  - 所有法条引用标注 `[未经校验]`
  - 不阻断写作流程
  - W4 阶段 D1 维度自动跳过

## 五、写作展开序列

### 5.1 案情事实涵摄

每个大纲要素展开前，先完成事实映射：
```
大纲要素：[要素名称]
对应事实：[从 W1 时间线/关键数字中提取的具体事实]
映射方式：[直接对应 / 推断 / 事实不足]
```

### 5.2 三层展开规则

遵循 `inner-structure.md` 定义：

#### 层一：定义层

- 给出该构成要件要素的学理定义
- 引用来源标注脚注
- 若存在多种定义 → 核心争议要素列举主要学说

#### 层二：涵摄层

- 将案情事实代入定义
- 逐要素对照分析
- 核心争议要素：展开学说争论 + 分析适用后果差异

#### 层三：结论层

- 得出该要素是否满足的结论
- 一句话表述，不重复论证

### 5.3 比例原则适用

在以下场景适用比例原则分析：
- 正当防卫的必要性判断
- 紧急避险的利益衡量
- 构成要件的限缩解释论证

展开格式：
```
适当性：[分析]
必要性：[分析]
狭义比例性：[分析]
```

### 5.4 法律解释方法适用

核心争议要素中，按以下顺序展开解释：
1. 文义解释
2. 体系解释
3. 历史解释
4. 目的解释

## 六、素材三级获取

### Tier 1：用户知识库

- 来源：用户配置中注册的刑法评注/教科书知识库
- 调用方式：通过对应 skill 检索
- 脚注格式：`[评注]` 或 `[释义]`
- 示例：`实质解释论代表立场认为，盗窃罪中的"占有"应理解为...[评注：参考文献②，p.1023]`

### Tier 2：案例与联网搜索

- 来源：MCP 案例检索 + WebSearch
- 调用条件：Tier 1 无法覆盖，或需要实务印证
- 脚注格式：`[案例]` 或 `[联网]`
- 示例：`最高人民法院指导案例第X号确认...[案例：指导案例X号]`

### Tier 3：自主解释

- 来源：AI 基于法学方法论自行展开
- 调用条件：Tier 1 + Tier 2 均无法覆盖
- **无脚注标注**
- 必须展示解释方法运作过程
- 示例：直接展开四步解释法，不引用外部权威

### 获取优先级铁律：
```
Tier 1 可用 → 必须使用 Tier 1
Tier 1 不可用 → 尝试 Tier 2
Tier 2 不可用 → 使用 Tier 3（标明为自主分析）
```

## 七、涵摄段落呈现

### 统一四步格式：

```markdown
**[要素名称]**

文义起点：
[法条/定义的文义范围界定]

体系协调：
[与相关条文的体系关联与一致性分析]

历史/目的支撑：
[立法背景 + 规范目的对解释方向的支撑]

结论：
[该要素是否满足 + 简要理由]
```

### 适用范围：
- 核心争议要素：完整四步展开
- 非争议要素：可简化为"定义 → 涵摄 → 结论"三步

## 八、法条引用格式

### 引用样式：

```markdown
> 《中华人民共和国刑法》第二百六十四条：
> "盗窃公私财物，数额较大的，或者多次盗窃、入户盗窃、
> 携带凶器盗窃、扒窃的，处三年以下有期徒刑、拘役或者管制，
> 并处或者单处罚金；……"
>
> [校验：2026-06-15，现行有效]
```

### 校验标签分类：

| 标签 | 含义 | 触发条件 |
|------|------|----------|
| `[校验：日期，现行有效]` | 条文已通过 MCP 验证 | 正常校验通过 |
| `[校验失败：原因]` | MCP 返回异常 | 条文不存在/已废止 |
| `[语义差异：说明]` | MCP 文本与引用不一致 | 内容比对发现出入 |
| `[未经校验]` | L2 降级模式 | MCP 不可用 |

## 九、写作进度标记

### 每个参与人写作完成后，输出进度标记：

```markdown
---
## 写作进度

| 写作单元 | 参与人 | 状态 | 完成时间 |
|----------|--------|------|----------|
| WU-1 | 甲 | ✓ 完成 | [时间] |
| WU-2 | 乙 | ◯ 进行中 | — |
| WU-3 | 丙 | — 待开始 | — |

当前进度：2/3 单元完成
---
```

### 进度状态定义：
- `✓ 完成`：该单元全部罪名检视论证完毕
- `◯ 进行中`：正在写作中
- `— 待开始`：尚未启动
- `✗ 阻断`：遇到无法解决的问题，需用户介入

## 十、竞合处理

### 位置（两种模式）：

**模式 A（默认）**：每位参与人全部罪名检视完毕后，在 W3 单元末尾独立小节处理。

**模式 B（可选）**：竞合分析剥离至 W4 合并阶段作为独立步骤。适用条件：参与人 ≥ 4 或子代理预估字数已接近上限时，由主编排 agent 决定是否启用。模式 B 的优势是减轻 W3 负担、确保竞合分析基于完整检视结论、避免跨子代理重复分析。

### 共犯关系基线引用：

涉及共同犯罪的案件，应先参照 `methodology/joint-crime-framework.md` 建立共犯关系基线，后续参与人直接引用基线而非重复论证。

### 展开结构：

```markdown
### [参与人] — 竞合论

1. 事实单元内竞合
   - [罪名A] 与 [罪名B]：想象竞合 / 法条竞合
   - 处理方式：[从一重处断 / 特别法优先 / ...]

2. 跨事实单元竞合
   - [罪名C]（单元1）与 [罪名D]（单元2）：实质竞合
   - 处理方式：[数罪并罚]

3. 最终结论：
   [参与人] 构成 [最终认定的罪名列表]，[竞合处理方式]
```
