# W4：差异核验与输出

> 本文件定义工作流第四阶段：对 W3 产出的鉴定论证文本进行三维核验，
> 执行一次性修订，经用户预审后生成终稿并保存全部副产物。

---

## 一、触发条件

- W3 全部写作单元状态为 `✓ 完成`
- 写作进度标记显示 100% 完成
- 若存在 `✗ 阻断` 单元 → 不得进入 W4，需先解决

---

## 二、三维核验

### D1：增量法条核验

**目的**：确保所有法条引用均已通过法律检索 MCP 校验、时效为现行有效。

**校验源**：
- `verification-log.md` + 法律检索 MCP `${LEGAL_SEARCH.get_article}`（动态校验现行有效性，部分服务商会自动追最新修正版）

**执行步骤**：

1. 扫描 W3 产出全文，提取所有法条引用（含具体条号 + 司法解释 doc_no）

2. 对每一条法条引用执行法律检索 MCP 终末复核：

   ```
   qw_mcp_call:
     toolName: ${LEGAL_SEARCH.get_article}
     arguments:
       title: "中华人民共和国刑法"               # 按 ${title_format} 契约
       number: "<按 ${number_format} 契约的条号>"
   ```

   判定规则：
   - 返回 `${timeliness_field}` 等于 `${timeliness_active_value}` 且 `article` 与 W3 引用文字一致 → `[D1-✓]`
   - 返回 `article` 与 W3 引用文字差异 → `[D1-修订]`，用服务商返回原文替换
   - 返回 `${timeliness_field}` 不等于 `${timeliness_active_value}` 或条号不存在 → `[D1-硬错]`，整条引用必须删除或替换

3. 对司法解释引用执行 `${LEGAL_SEARCH.search_article}` 复核（用 doc_no 或罪名+要素关键词），过滤 `${timeliness_field}` 等于 `${timeliness_active_value}`

4. 比对 `verification-log.md`：
   - 已有记录且通过 → 标记 `[D1-✓]`（可省略复核调用，节省成本）
   - 已有记录但失败 → 检查是否仍被引用（不应引用，标记为修订项）
   - 无记录 → 新增法律检索 MCP 校验，写入日志

5. 输出 D1 核验报告：

```markdown
# D1 核验报告：法条校验

| 法条 / 司法解释 | ${timeliness_field} 状态 | verification-log 状态 | 正文引用位置 | 核验结果 |
|------|------|----------------------|--------------|----------|
| 《刑法》第264条（2023修正） | 现行有效 | ✓ 已校验 | 甲-盗窃-客观TB | D1-✓ |
| 法释〔2013〕8号第1条 | 现行有效 | ✓ 已校验 | 甲-盗窃-数额标准 | D1-✓ |
| 《刑法》第20条 | 现行有效 | — 未记录 | 甲-盗窃-违法性 | [新增校验] ✓ |
| 法释〔1998〕4号 | **废止或失效** | ✗ 校验失败 | 甲-盗窃-数额（旧标准） | **D1-硬错** |

总计引用法条：X 条
已校验通过：Y 条
新增校验：Z 条
需修订：W 条
```

**L2 降级模式**：D1 维度自动跳过，报告标注 `[D1 因 L2 降级跳过]`。

**增量校验优化**：若 `verification-log.md` 中已有校验记录且日期在同一天内，D1 可完全跳过 MCP 调用，仅做"日志审计"——逐条比对日志记录与正文引用是否一致。仅当日志缺失或校验日期非当日时，才执行 MCP 复核调用。

---

### D2：大纲覆盖与事实保真

**目的**：确保大纲每一要素均已展开，且论证中的事实引用忠实于 W1 原始提取。

**执行步骤**：

1. **大纲覆盖检查**：
   - 逐项比对闸二确认大纲与 W3 正文
   - 每个大纲节点必须在正文中有对应展开
   - 检查核心争议要素是否达到 ≥500 字
   - 检查非争议要素是否在 100-200 字范围

2. **事实保真检查**：
   - 提取 W3 正文中所有事实陈述
   - 逐条比对 W1 五节提取结果
   - 标记：`[忠实]` / `[偏离]` / `[新增]`
   - 偏离或新增事实 → 标记为修订项

3. 输出 D2 核验报告：

```markdown
# D2 核验报告：覆盖与保真

## 覆盖检查
| 大纲节点 | 正文位置 | 字数 | 达标 |
|----------|----------|------|------|
| 甲-《刑法》第264条-行为方式(**争议**) | §I.1.(3) | 620字 | ✓ |
| 甲-《刑法》第264条-故意 | §I.2.(1) | 150字 | ✓ |
| ... | ... | ... | ... |

未覆盖节点：[列表，若有]
字数不达标节点：[列表，若有]

## 事实保真检查
| 正文事实陈述 | W1 来源 | 保真状态 |
|--------------|---------|----------|
| "甲于X日转账Y元" | §2 时间线 + §3 数字表 | 忠实 |
| "甲知晓乙的身份" | [未见于 W1] | **偏离/新增** |

需修订项：[数量]
```

## 三、一次性修订

### 修订原则：
- **单次通过**，不循环
- 优先级排序：P1 → P2 → P3

### 修订执行：

#### P1：法条准确性修订

基于 D1 报告中的修订项：
- 校验失败的法条 → 删除引用或替换为正确条文
- 新增校验失败 → 同上处理
- 修订后更新 `verification-log.md`

#### P2：覆盖与保真修订

基于 D2 报告中的修订项：
- 未覆盖大纲节点 → 补充展开（遵循原字数标准）
- 事实偏离 → 修正为 W1 原始事实
- 事实新增（无 W1 依据）→ 删除或标注 `[案情未明确]`

#### P3：格式修订

基于 D3 报告中的修订项：
- 逐项修正格式问题
- 术语不一致 → 全文统一替换
- 标题层级错误 → 调整

### 修订约束：
- 每个修订项仅处理一次
- 修订完成后不再返回核验（防止无限循环）
- 若某修订项无法完成（如事实确实存在歧义）→ 保留并标注，交由用户预审决定

## 四、竞合处理专节

### 定位

一次性修订完成后、全案结论组装之前，执行独立的竞合处理分析。此步骤由主流程引擎（而非子代理）执行，确保基于全部 W3 产出而非局部视角。

### 执行步骤

1. **汇总全部已成立的罪名**：从 W3 各写作单元的中间结论中提取所有 ✓ 成立的罪名。

2. **逐人分析竞合关系**：

   按以下顺序依次判断：

   | 竞合类型 | 判断标准 | 处理方式 |
   |----------|----------|----------|
   | 法条竞合 | 同一行为同时满足具有特别/补充/吸收关系的多个条文 | 特别法优于普通法 / 重法优于轻法 |
   | 想象竞合 | 一行为触犯数罪名 | 从一重罪处断 |
   | 实质竞合 | 数行为触犯数罪名 | 数罪并罚（§69-§71） |
   | 同种数罪 | 数行为触犯同一罪名 | 通常不并罚，累计数额或作为量刑情节 |

3. **输出最终的罪数结论与量刑框架**：

   ```markdown
   ## 竞合处理结论
   
   ### [参与人甲]
   
   | 成立罪名 | 竞合关系 | 处理结论 |
   |----------|----------|----------|
   | 罪名A (§X) | 与罪名B 想象竞合 | 择一重，以罪名A 论处 |
   | 罪名B (§Y) | ↑ | ↑ |
   | 罪名C (§Z) | 独立 | 与上述罪名实质竞合，数罪并罚 |
   
   最终定罪：罪名A + 罪名C（数罪并罚）
   ```

4. **与大纲竞合预览的一致性校验**：比对闸二大纲中的竞合关系前置标注（`2b-outline.md §2.5`），确认 W3 实际分析结果与预览一致或标注差异原因。

## 七、终稿输出

### 7.1 输出格式选择

根据 `.user-config.md` 中的报告格式配置：

#### 学术 Word 格式：

输出链路（按优先级尝试）：
```
优先：cx-md2word 技能 → .docx 文件
备选：docx 技能 → .docx 文件  
兜底：Pandoc CLI → .docx 文件
```

Word 格式要求：
- A4 页面尺寸
- 页边距：上下 2.54cm，左右 3.17cm
- 正文字体：宋体 12pt / Times New Roman 12pt（外语术语）
- 标题字体：黑体，各级大小递减
- 脚注：页面底部，小五号宋体
- 页眉：案件简称
- 页脚：页码

#### Markdown 原始格式：

- 直接输出完整 Markdown 文件
- 保留全部标题层级与格式标记
- 脚注使用 Markdown 脚注语法 `[^1]`

### 7.2 输出文件命名

```
[日期]-[案件简称]-鉴定式分析报告.[docx/md]
```

示例：`2026-06-15-甲乙盗窃案-鉴定式分析报告.docx`

### 保存目录结构：

```
[工作目录]/
├── case-summary.md
├── outline.md
├── verification-log.md
├── final-output.[docx/md]
├── drafts/
│   ├── draft-甲.md
│   ├── draft-乙.md
│   └── ...
└── reports/
    ├── report-D1.md
    ├── report-D2.md
    └── report-D3.md
```

### 保存时机：
- W1 完成 → 保存 `case-summary.md`
- 闸二确认 → 保存 `outline.md`
- W3 各单元完成 → 保存对应 `draft-[参与人].md`
- W4 核验完成 → 保存三份报告
- 终稿生成 → 保存 `final-output`
- `verification-log.md` 全程增量更新
