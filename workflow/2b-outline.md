# W2 闸二：检视大纲与独立确认

> 本文件定义工作流第二阶段的第二道闸门：基于闸一确认的候选罪名清单，
> 按三阶层犯罪论体系展开检视大纲，标记核心争议，独立确认后进入写作。

---

## 一、硬边界

### 1.1 前置条件

- **闸一必须独立确认通过**方可启动闸二
- 若闸一未确认 → 拒绝执行，提示："请先完成闸一确认"
- 本阶段产物 = **仅大纲**，不含涵摄论证

### 1.2 本阶段禁止事项

| 禁止内容 | 说明 |
|----------|------|
| 涵摄段落 | 属于 W3 产物 |
| 结论判断 ("成立"/"不成立") | 属于 W3 小结 |
| 超过 3 行的论证展开 | 大纲为结构骨架，非正文 |
| 对法条的实质解释 | 属于 W3 素材展开 |

### 1.3 新增条文校验

- 大纲展开过程中若涉及闸一未列入的新条文（如阻却事由条款）
- 必须立即通过 MCP 进行形式校验
- 校验通过 → 追加写入 `verification-log.md`
- 校验失败 → 不得列入大纲，标注 `[待校验]`

| 4 | `methodology/crime-type-library/`（按需叠加） | 精化检视程式（故意/过失/不作为/共同犯罪/结果加重/认识错误/竞合 7 类） |

各模块按上表顺序串行执行，输出物逐级累积，最终汇集为闸二大纲。

### 2.1 单一罪名展开格式

对每一个候选罪名，按以下结构生成大纲：

```markdown
### [参与人] — [罪名] (§[条文])

(一) 构成要件符合性

   1. 客观构成要件
      (1) 行为主体 — §[Art.X] — [MCP ✓/✗]
      (2) 行为客体 — §[Art.X] — [MCP ✓/✗]
      (3) 行为方式 — §[Art.X] — [MCP ✓/✗] —— **核心争议**
      (4) 因果关系 — 条件公式
      (5) 客观归责 — [如涉及]
      (6) 结果 — §[Art.X] — [MCP ✓/✗]
      ...

   2. 主观构成要件
      (1) 故意 — 直接故意/间接故意
      (2) 特殊主观要素 — [如涉及] —— **核心争议**
      ...

(二) 违法性

   A. 正当防卫 — 《刑法》第20条 — [MCP ✓/✗] — [如相关展开]
      - 防卫情势
      - 防卫行为
      - 防卫意思
   B. 紧急避险 — 《刑法》第21条 — [如相关展开]
   C. 被害人同意 — [如相关展开]
   D. 其他阻却事由（如依法令或职务的行为：警察执行公务、医生紧急救治等） — [如相关展开]

(三) 有责性

   A. 责任能力 — 《刑法》第17条/第17条之一/第18条/第19条 — [MCP ✓/✗]
      - 年龄
      - 精神状态
   B. 违法性认识可能性 — [如相关展开]
   C. 期待可能性 — [如相关展开]

(四) 小结：[罪名] ✓成立 / ✗不成立 / ?待论证
```

### 2.5 竞合关系前置标注

在每位参与人的大纲末尾，除了标注"竞合处理预览"外，增加以下两类竞合标注：

**(a) 同单元竞合标注**：

如果同一行为单元涉及两个以上罪名，在各自罪名的提纲中标注：

```markdown
   [罪名A] 的提纲中追加：
   → 【竞合提示】与 [罪名B] 存在 [想象竞合/法条竞合] 关系，
     在检视 [罪名A] 时应同步考虑
```

**(b) 跨单元竞合标注**：

不同行为单元之间的实质竞合关系预览：

```markdown
   ## [参与人] — 竞合关系预览
   
   | 罪名对 | 竞合类型 | 处理方向 |
   |--------|----------|----------|
   | 罪名A ↔ 罪名B | 想象竞合 | 择一重 |
   | 罪名C ↔ 罪名D | 实质竞合 | 数罪并罚 |
```

此标注确保 W3 写作时各罪名的 subagent 能同步考虑竞合关系，而非推迟到末尾才发现竞合问题。

### 2.2 展开规则

- 每个要素节点必须标注：
  - 对应法条（若有）
  - MCP 校验状态：`[MCP ✓]` 已校验通过 / `[MCP ✗]` 校验失败 / `[MCP —]` 无需校验
  - 争议标记（若存在）
- 结果犯必须展开因果关系 + 客观归责
- 行为犯可省略因果关系节点
- 不作为犯需单独展开保证人地位
- 过失犯需调整主观要件为注意义务违反

### 2.3 未遂形态补充

若事实单元中行为未达既遂，在构成要件符合性之后插入：

```markdown
   *. 未遂检视
      (1) 未遂可罚性 — 《刑法》第23条（中国法对故意犯罪原则上承认未遂的可罚性，但举动犯等特殊罪名除外；第23条仅规定"可以从轻或者减轻处罚"，无关于"不能犯"的明文规定，"不能犯不可罚"系学理通说）
      (2) 犯罪决意
      (3) 直接着手
      (4) 未遂类型：障碍未遂/中止未遂（《刑法》第24条："没有造成损害的，应当免除处罚；造成损害的，应当减轻处罚"）
```

### 2.4 共犯形态补充

对共犯参与人，在正犯检视后追加：

```markdown
   *. 共犯检视
      (1) 正犯行为存在 — [从属性审查；中国《刑法》第25-29条规定共同犯罪体系，限制从属性系学理处理]
      (2) 共犯行为
         - 教唆行为（教唆犯，《刑法》第29条） / 从犯行为（含次要作用与辅助作用两类，《刑法》第27条） / 胁从犯行为（《刑法》第28条）
      (3) 双重故意（注：中国法上过失共同犯罪不以共同犯罪论处，《刑法》第25条第2款）
```

## 三、核心争议标记

### 标记格式

在大纲要素行末尾添加：`—— **核心争议**`

### 标记标准

符合以下任一条件即标记为核心争议：
1. 案情事实对该要素的满足存在两种以上合理解释
2. 学界对该要素的认定标准存在显著分歧（如因果关系理论）
3. 司法实践对该要素的适用存在典型案例冲突
4. 案情给出的信息不足以直接判断该要素

### 核心争议的下游效果

- W3 对标记为核心争议的要素：展开 ≥500 字深度论证
- W3 对未标记要素：简要涵摄即可（100-200 字）
- 核心争议数量建议：每个罪名 1-3 个为宜

## 四、闸二确认

### AskUserQuestion 参数：

- question: "检视大纲是否准确？请确认或选择需要调整的方面。"
- header: "闸二确认"
- multiSelect: true
- options:
  1. label: "大纲确认无误" / description: "检视结构完整准确，进入 W3 写作阶段"
  2. label: "调整展开范围" / description: "指定需要增减的检视要素（如跳过某阻却事由）"
  3. label: "调整争议标记" / description: "修改核心争议的标记位置或数量"
  4. label: "激活知识库" / description: "指定 W3 写作时调用的评注/教科书知识库"

### 闸二确认的衍生选择：

与闸一不同，闸二确认允许附带以下衍生问题：
- 是否展开所有阻却事由（即使案情未提示）
- 是否对共犯参与人完整展开三阶层（vs 简略从属性检视）
- 知识库优先级排序（当多个知识库可用时）

### 确认后处理：
- 用户确认 → 锁定大纲版本，进入 W3
- 大纲锁定后，W3 不得自行修改结构（除非发现方法论错误）

## 六、违例自检

### 以下 4 项指标表明 AI 混淆了闸二与 W3 内容：

| 编号 | 违例指标 | 检测方法 |
|------|----------|----------|
| V1 | 大纲某要素下出现 ≥4 行连续论证段落 | 行数统计 |
| V2 | 出现"因此"、"综上"、"应当认定为"等结论性连接词 | 关键词匹配 |
| V3 | 引用具体案例或学说展开分析 | 脚注/引用扫描 |
| V4 | 小结部分给出确定性结论（✓/✗）而非占位符（?待论证） | 结尾字符检查 |

### 自检规则：
- 大纲生成完毕后、呈现给用户之前执行
- 小结栏原则上使用 `?待论证` 占位
- 仅当案情事实极其明确且无任何争议时，方可预填 ✓/✗
- 违例触发 → 自动精简为骨架结构后重新输出

## 七、与下游的连接

### 输出物清单：
1. 完整检视大纲（结构化 Markdown）
2. 核心争议列表（带标记位置）
3. 并行写作单元注册表
4. 更新后的 `verification-log.md`

### 流转至 W3：
- 大纲 + 校验日志 + 写作单元注册 → 作为 `3-drafting.md` 的输入
- W3 严格按照大纲结构展开，不得自行增删章节
- 核心争议标记指导 W3 的深度分配
